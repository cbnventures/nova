# 快速开始

Foundry 是一台面向开发者意图的编译器。你在一份 `.grain` 清单中描述工作空间应有的样子，Foundry 便会锻造出完整的开发环境——目录结构、依赖图、构建流水线、检查规则与测试脚手架。

不再有配置文件。不再有手工接线。一份清单进来，整套环境出去。

## Foundry 做了什么

Foundry 读取 `.grain` 清单，并生成多包工作空间在构建、检查、测试与部署所需的一切。你不必再维护跨包的几十份配置文件，只需维护一份声明你意图的文件。

| 工具           | 用途                              |
|--------------|---------------------------------|
| **Anvil**    | 核心 CLI——读取 Blueprint 清单并锻造工作空间。 |
| **Tongs**    | 依赖图解析器与可视化工具。                   |
| **Quench**   | 增量构建缓存——只重建变更过的部分。              |
| **Bellows**  | 跨工作空间的并行任务执行器。                  |
| **Crucible** | 测试套件生成器——从类型签名生成测试脚手架。          |
| **Slag**     | 死代码与未使用依赖的检测工具。                 |

## 快速上手

通过 Spark 全局安装 Foundry：

```bash title="安装 Foundry"
spark install foundry
```

在项目根目录创建一份最小的 `.grain` 清单：

```text title="project.grain"
workspace "my-app" {
  lang   = "alloy"
  warden = ["strict"]

  packages {
    core { type = "library" }
    api  { type = "service", depends = ["core"] }
  }
}
```

锻造工作空间：

```bash title="运行你的第一次锻造"
foundry ignite
```

Foundry 会读取清单、解析依赖图、生成 Warden 规则、搭建测试文件，并写出每个包所需的配置文件。

:::tip
运行 `foundry ignite --dry-run` 可在不向磁盘写入任何文件的情况下，预览 Foundry 将生成的内容。这是提交前验证清单最快的方式。
:::
