我们见过那种过了日历上每一场审计、却仍然被攻陷的组织；我们也见过那种连一场审计都过不了、但安全态势能让某些国家级对手停下来想一想的组织。这两件事并不是同一件事。

## 那道差距

合规是一份清单，安全是一种态势。清单告诉你某项控制是否存在；态势告诉你它是否真在工作。

以磁盘加密为例。合规框架问的是："端点设备是否加密？"答案是是或否。安全态势问的是："端点设备是否加密、加密算法是否当前、密钥是否在过去 90 天内轮换过、是否有任何设备上报过解密失败？"答案是一个连续谱。

| 合规的问题      | 安全的问题                           |
|------------|---------------------------------|
| 设备是否加密？    | 使用 AES-256 与使用已过时算法的设备比例分别是多少？  |
| 是否启用了 MFA？ | MFA 提示到批准的平均时长是多少？是否有人在敷衍地一键通过？ |
| 是否保留访问日志？  | 是否能仅凭日志数据完整还原一次会话？              |
| 是否有事件响应预案？ | 上一次以仿真入侵进行实战演练是什么时候？            |
| 是否打了补丁？    | 从 CVE 披露到全设备群修复的平均时长是多少？        |

这张表每一行，都是"通过审计"与"真的安全"之间的差距。

## Spyglass 如何弥合这道差距

Spyglass 是 Sentinel 的审计与取证引擎。它为每一次访问决策、每一次策略评估、每一次连接事件都保留 7 年的不可篡改日志。但保留不是重点，还原才是。

合规视角告诉审计员"控制存在"；安全视角告诉安全团队"到底发生了什么"。两者来自同一份数据，差别在于你问的是什么问题。

## 合规无法度量的三件事

**1. 反应速度。** 合规框架不度量你发现并控制威胁的速度。Spyglass 记录每一次策略评估的时间戳，我们能把"从异常到撤销"的平均时长度量到毫秒。

**2. 横向移动暴露面。** 没有任何审计清单会问"一个被攻陷的服务账号在撞到分段边界之前能跳几步？"。Rampart 区域拓扑与 Parapet 仿真持续回答这个问题。

**3. 信任衰减。** 八小时前完成认证的用户，与三十秒前完成认证的用户，风险并不相同。合规框架把认证当成二元事件；Sentinel 把它当成一份在贬值的资产——态势评分会衰减、会话时长会累积、Watchtower 每 90 秒重新评估整个画面。

## 我们的建议

不要在合规与安全之间二选一。先把安全态势建起来，再从中抽取合规证据。

Sentinel 就是按这种模式设计的。Spyglass 用驱动安全运营的同一份数据流生成合规报告。审计证据是"把安全做对"的副产品，而不是一条与工程时间相互争抢的并行工作流。

```bash title="从实时数据生成合规证据"
sentinel spyglass export --framework hipaa --period 2025-Q3
sentinel spyglass export --framework pci-dss --period 2025-Q3
sentinel spyglass export --framework iso27001 --period 2025-Q3
```

三条命令，三套框架，同一份底层数据。审计早已完成——你只是把它格式化而已。
