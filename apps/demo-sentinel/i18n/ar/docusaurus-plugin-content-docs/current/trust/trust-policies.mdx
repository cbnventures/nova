# سياسات الثقة

سياسات الثقة هي الوحدة الأساسية للتحكّم في الوصول داخل Sentinel. تُحدّد السياسة الشروط التي يجب أن تتحقّق لكي يُمنح الوصول، وما يحدث حين لا تعود تلك الشروط مستوفاة. تُكتب السياسات بصيغة `.grain`، وتخضع للتحكّم بالإصدارات، ويُقيّمها Watchtower كلّ 90 ثانية.

## قواعد السياسة

تتألّف كلّ سياسة من تصريح، وكتلة شروط، ومعالج إخفاق:

```text title="policies/api-access.grain"
policy "api-cluster-access" {
  resource = "api-cluster-east"
  effect   = "allow"
  priority = 100

  conditions {
    device.posture   >= 85
    user.mfa         = true
    user.role        = ["engineer", "sre"]
    network.location = ["office", "vpn"]
    session.age      <= 7200
  }

  on_failure {
    action  = "revoke"
    notify  = "security-ops"
    log     = "spyglass"
    step_up = "mfa"
  }
}
```

حقل `effect` إمّا `allow` أو `deny`. وتسبق سياسات `deny` دائماً سياسات `allow` عند مستوى الأولوية ذاته.

## أنواع الشروط

| الشرط               | العامل          | القيم                         | الوصف                                               |
|---------------------|-----------------|-------------------------------|-----------------------------------------------------|
| `device.posture`    | `>=`, `<=`, `=` | 0-100                         | درجة الوضع الأمني المركّبة من Garrison.             |
| `device.encryption` | `=`             | `true`, `false`               | حالة تشفير القرص.                                   |
| `device.os`         | `=`             | مصفوفة من أسماء أنظمة التشغيل | أنظمة التشغيل المسموح بها.                          |
| `user.mfa`          | `=`             | `true`, `false`               | تسجيل المصادقة متعدّدة العوامل.                     |
| `user.role`         | `=`             | مصفوفة من أسماء الأدوار       | الأدوار المطلوبة (أيّ تطابق).                       |
| `user.department`   | `=`             | مصفوفة من أسماء الأقسام       | الأقسام المطلوبة (أيّ تطابق).                       |
| `network.location`  | `=`             | مصفوفة من وسوم المواقع        | مواقع الشبكة المسموح بها.                           |
| `network.ip_range`  | `=`             | مصفوفة من كتل CIDR            | نطاقات IP المصدر المسموح بها.                       |
| `behavior.score`    | `>=`, `<=`, `=` | 0-100                         | درجة شذوذ السلوك من Watchtower.                     |
| `session.age`       | `<=`            | ثوانٍ                         | المدّة القصوى للجلسة قبل إعادة المصادقة الإجباريّة. |
| `time.window`       | `=`             | تعبير cron                    | النافذة الزمنية التي تكون فيها السياسة فعّالة.      |

يجب استيفاء جميع شروط السياسة في وقت واحد. لا يوجد منطق OR ضمن سياسة واحدة، استخدم تركيب السياسات للحصول على دلالات OR.

:::info ترتيب التقييم
حين تنطبق سياسات متعدّدة على المورد ذاته، يُقيّمها Sentinel بترتيب الأولوية (الأرقام الأدنى أولاً). تُحدّد أوّل سياسة مطابقة قرار الوصول. وسياسات الرفض في الأولوية ذاتها لسياسة سماح تفوز دائماً.
:::

## تركيب السياسات

يمكن تركيب السياسات عبر الوراثة والتسلسل. ترث السياسة الفرعية جميع الشروط من السياسة الأم، ويمكنها إضافة شروط محدّدة أو تجاوزها:

```text title="policies/base-access.grain"
policy "base-access" {
  resource = "*"
  effect   = "allow"

  conditions {
    device.posture >= 70
    user.mfa       = true
  }
}
```

```text title="policies/production-access.grain"
policy "production-access" {
  // highlight-next-line
  extends  = "base-access"
  resource = "production-*"
  priority = 50

  conditions {
    // highlight-start
    device.posture   >= 90
    network.location = ["office"]
    session.age      <= 3600
    // highlight-end
  }
}
```

ترث سياسة `production-access` الشرط `user.mfa = true` من `base-access`، وتتجاوز عتبة الوضع الأمني من 70 إلى 90. كما تُضيف قيوداً على الموقع وعمر الجلسة لا تمتلكها السياسة الأم.

## دورة التقييم

يُقيّم Watchtower جميع الجلسات النشطة مقابل سياساتها المعمول بها كلّ 90 ثانية:

1. يُبلّغ Garrison عن الوضع الأمني الحالي لكلّ جهاز مسجّل.
2. يجمع Watchtower بيانات الوضع الأمني، وهوية المستخدم، والسياق الشبكي، والإشارات السلوكية.
3. لكلّ جلسة نشطة، يحلّ Watchtower سلسلة السياسات المعمول بها (الوراثة + الأولوية).
4. تُقيَّم جميع الشروط. وإن أخفق أيّ شرط، يعمل معالج `on_failure`.
5. يُسجّل Spyglass التقييم الكامل، سواء نجح أم أخفق، مع الطوابع الزمنية وقيم الإشارات.

<Details>
<summary>المرجع الكامل لعوامل الشروط</summary>

**العوامل العددية** (`>=`, `<=`, `=`)، تُستخدم لدرجات الوضع الأمني، ودرجات السلوك، وعمر الجلسة. تُقارن القيم بوصفها أعداداً صحيحة.

**عامل المساواة** (`=`)، يُستخدم للقيم البوليانية (MFA، التشفير)، والمصفوفات (الأدوار، المواقع، أسماء أنظمة التشغيل)، ونطاقات CIDR. تستخدم شروط المصفوفات دلالات "أيّ تطابق"، يتحقّق الشرط إن طابقت القيمة الفعلية أيّ عنصر في المصفوفة.

**عامل cron** (`=` مع تعبير cron)، يُستخدم لـ `time.window`. يتحقّق الشرط فقط خلال النافذة الزمنية المُحدّدة بتعبير cron. وخارج النافذة، يُخفق الشرط ويعمل معالج `on_failure`.

**النفي**، استخدم البادئة `!` أمام أيّ شرط لعكسه، `!network.location = ["public-wifi"]` يمنع الوصول من شبكات الواي فاي العامّة.

</Details>
