# Manifesto

Lantern exists because shipping software in private is a special kind of darkness. Things that are built but never described do not really exist. We believe documentation is illumination — the work of turning *something happening* into *something visible*. A changelog is a window. A setup guide is a porchlight. A reference page is a streetlamp on a long road.

## Documentation as Illumination

A page is a small light. Lit pages add up the way porchlights add up — until a road that was dark is a road that can be walked. We aim to write pages that turn on for the reader, not pages that explain to ourselves that we already know things.

## Warmth Before Rigor

Rigor without warmth reads like a security guard reading the door policy. Warmth without rigor reads like a guess. We try for both, and when the two cannot fit on the same line we pick the one the reader will stay for.

## The First Sentence

The hardest sentence in any document is the first one after the title. Most everything good downstream comes from getting it right. Most everything wrong downstream comes from leaving it generic.

### Writing in the dark

A page written for someone who already knows is not really a page. It is a private note pinned to a door. We try to leave the door open.
