## خط أنابيب الترحيل

Relay هو محرك العمليات بالجملة لدى Signal. يتولى إنشاء الروابط بالجملة وتعديلها وترحيلها عبر صيغة مهام تعريفية:

```json title="مهمة ترحيل Relay"
{
  "job": "migrate",
  "source": {
    "provider": "bitly",
    "apiKey": "env:BITLY_API_KEY",
    "filter": {
      "createdAfter": "2022-01-01",
      "status": "active"
    }
  },
  "target": {
    "domain": "go.fintech.example",
    "preserveSlugs": true,
    "attribution": {
      "campaign": "migration-q4",
      "channel": "legacy"
    }
  },
  "options": {
    "dryRun": false,
    "concurrency": 50,
    "redirectChain": true
  }
}
```

خيار `redirectChain` حاسم. يُعيد ضبط الرابط القصير القديم لإعادة التوجيه عبر Beacon قبل الوصول إلى الوجهة النهائية. يتم توجيه المستخدمين الذين ينقرون رابطاً قديماً بسلاسة عبر النظام الجديد، ويُسنَد النقر في تحليلات Beacon من تلك اللحظة فصاعداً.

## الأداء على نطاق واسع

قِسنا أداء Relay عبر أحجام مهام مختلفة:

| الروابط | المدة    | الإنتاجية  | معدل الخطأ |
|---------|----------|------------|------------|
| 1,000   | 4 ثوانٍ  | 250 رابط/ث | 0.00%      |
| 10,000  | 38 ثانية | 263 رابط/ث | 0.00%      |
| 50,000  | 3د 12ث   | 260 رابط/ث | 0.01%      |
| 140,000 | 11د 04ث  | 211 رابط/ث | 0.02%      |

تبقى الإنتاجية متّسقة حتى نحو 50 ألف رابط. وفيما يتجاوز ذلك، يصبح تحديد المعدل على واجهات مزود المصدر هو العنق المضيق، لا Relay نفسه. كان معدل الخطأ عند 140 ألف رابط 0.02%، ما يساوي 28 رابطاً احتاجت مراجعة يدوية بسبب عناوين URL وجهة مشوّهة في بيانات المصدر.

## التعديل بالجملة

الترحيل حالة استخدام واحدة. أكثر عمليات Relay شيوعاً هي التعديل بالجملة، تحديث آلاف الروابط مرة واحدة عند تغيّر حملة أو دوران نطاق:

- **تدوير النطاقات**: نقل جميع الروابط من `go.old-brand.com` إلى `go.new-brand.com` في مهمة واحدة.
- **تعبئة الإسناد**: إضافة بيانات حملة إلى روابط قديمة أُنشئت دونها.
- **اكتساح الانتهاء**: ضبط تواريخ انتهاء على الروابط الأقدم من عتبة.
- **تحديث الوجهة**: إعادة توجيه كل الروابط المشيرة إلى URL متوقف إلى بديله.

تعمل كل عملية كمهمة ذرية بدعم تراجع كامل. إن فشلت مهمة في منتصفها، يُعيد Relay التغييرات الجزئية ويُبلِغ بدقة عن الروابط التي تعذّر معالجتها.

## لماذا تهم العمليات بالجملة

تعامل معظم أدوات إدارة الروابط الروابطَ ككائنات فردية. تُنشئها فرداً فرداً، وتعدّلها فرداً فرداً، وتحلّلها فرداً فرداً. يعمل هذا حين يكون لديك مئة رابط. ولا يعمل حين يكون لديك مئة ألف.

يتعامل Relay مع الروابط كمجموعة بيانات. تُعرّف العملية تعريفياً، ويتحقق Relay منها على كامل مخزون الروابط، ثم يُنفذها. الفارق بين إدارة الروابط فردياً وإدارتها بالجملة هو ذاته الفارق بين تحرير خلايا في جدول بيانات وتشغيل ترحيل SQL.
