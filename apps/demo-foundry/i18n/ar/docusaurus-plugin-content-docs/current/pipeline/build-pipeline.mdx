# خط إنتاج البناء

يبني Foundry مساحات العمل في خمس مراحل: الحلّ، وفحص الذاكرة، والترجمة، والربط، والتحقق. أداتان تقودان هذا الخط — Quench يتولى التخزين، وBellows يتولى التوازي.

## دورة حياة Forge

كل استدعاء لـ `foundry ignite` يتبع التسلسل ذاته:

1. **الحلّ.** يقرأ Tongs البيان ويبني رسم التبعيات.
2. **فحص الذاكرة.** يجزّئ Quench كل ملف مصدر وإصدار تبعية وتوجيه بيان. إن تطابقت التجزئة مع بناء سابق، تُتخطى الحزمة.
3. **الترجمة.** يُترجم Smelter ملفات المصدر لكل حزمة.
4. **الربط.** تُحلّ المراجع عبر الحزم وتُوصَل بدائل الاستيراد.
5. **التحقق.** يعمل Warden على الخرج المُترجَم. يشغّل Crucible الاختبارات المولّدة إن كان مُهيَّأً.

## تخزين Quench

Quench ذاكرة بناء موجّهة بالمحتوى. لا يستخدم طوابع وقت الملفات — فالطوابع غير موثوقة عبر الأجهزة وبعد عمليات التحكم في الإصدار. بدلًا من ذلك، يجزّئ Quench ثلاثة مدخلات:

| المدخل           | الوصف                                               |
|------------------|-----------------------------------------------------|
| محتوى المصدر     | SHA-256 لكل ملف في دليل `src/` للحزمة.              |
| إصدارات التبعيات | سلسلة الإصدار المحلولة لكل تبعية في رسم Tongs.      |
| توجيهات البيان   | قواعد Warden وإعداد Smelter والهدف من ملف `.grain`. |

إذا تطابقت التجزئات الثلاث جميعها مع بناء مخزَّن، يستعيد Quench القطع ويتخطّى الترجمة تمامًا.

```text title="Cache status output"
$ foundry quench status
  core       [HIT]  abc12f → cached 2m ago
  auth       [HIT]  def34a → cached 2m ago
  api        [MISS] ghi56b → source changed (src/routes/health.al)
  web        [HIT]  jkl78c → cached 14m ago
  gateway    [WAIT] depends on api (MISS), must rebuild
```

### الطَرقات الدافئة مقابل الباردة

الطَرق البارد يعمل بذاكرة فارغة. تُترجَم كل حزمة من الصفر. الطَرق الدافئ يعمل حين تتطابق معظم التجزئات — تُعاد بناء الحزم المتغيّرة ومعتمداتها السفلية فقط.

| حجم مساحة العمل | طَرق بارد  | طَرق دافئ |
|-----------------|------------|-----------|
| 10 حزم          | 8.1 ثانية  | 1.2 ثانية |
| 50 حزمة         | 14.7 ثانية | 1.8 ثانية |
| 100 حزمة        | 28.3 ثانية | 2.1 ثانية |
| 200 حزمة        | 52.6 ثانية | 3.4 ثانية |

### صيانة الذاكرة

قلِّم مُدخلات الذاكرة القديمة لاستعادة مساحة القرص:

```bash title="Remove entries older than 7 days"
foundry quench prune --older-than 7d
```

## توازي Bellows

يقرأ Bellows رسم تبعيات Tongs ويُجدول الحزم في موجات. الحزم المستقلة تُبنى في الوقت ذاته. الحزم المعتمدة تنتظر مسبقاتها.

<Tabs>
<TabItem value="local" label="التطوير المحلي" default>

```bash title="Run a parallel forge locally"
foundry ignite
```

يكتشف Bellows تلقائيًا عدد نوى CPU ويوزّع العمل وفقًا لذلك. يمكن التجاوز عبر متغيّر البيئة `FOUNDRY_PARALLELISM`.

</TabItem>
<TabItem value="ci" label="Conduit (CI)">

```bash title="Run a forge in Conduit"
foundry ignite --ci
```

في وضع CI، يُنتج Foundry خرج JSON مُهيكَلًا لدمج خطوط الأنابيب، ويُعطّل المطالبات التفاعلية، ويحترم متغيّر `FOUNDRY_CACHE_DIR` لأدلّة الذاكرة المشتركة.

</TabItem>
</Tabs>

### ترتيب التنفيذ

يُجمِّع Bellows الحزم في موجات بناءً على رسم التبعيات:

```text title="Bellows execution log"
$ foundry bellows build --verbose
  Wave 1: core (no dependencies)
  Wave 2: auth, cli, docs (depend on core)
  Wave 3: api, web (depend on core + auth)
  Wave 4: gateway (depends on api + auth)

  Total: 4 waves, 7 packages, 3.4s (warm)
```

الحزم في الموجة ذاتها تعمل بالتوازي. الموجات تُنفَّذ تسلسليًا. زمن البناء الإجمالي هو مجموع أطول حزمة في كل موجة — لا مجموع كل الحزم.

## استراتيجيات البناء

| الاستراتيجية | الأمر                              | حالة الاستخدام                  |
|--------------|------------------------------------|---------------------------------|
| طَرق كامل    | `foundry ignite`                   | توليد كل القطع من البيان.       |
| تزايدي       | `foundry ignite --incremental`     | إعادة بناء الحزم المتغيّرة فقط. |
| حزمة واحدة   | `foundry bellows build --only api` | بناء حزمة واحدة وتبعياتها.      |
| تشغيل جاف    | `foundry ignite --dry-run`         | معاينة الخرج دون كتابة ملفات.   |
