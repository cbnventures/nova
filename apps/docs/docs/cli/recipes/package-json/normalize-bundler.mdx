Normalize **types**, **module**, **sideEffects**, and **esnext** <Term title="build tool" to="/docs/quickstart/terminology#builds-and-tooling">bundler</Term> convention fields in <Term title="package" to="/docs/quickstart/terminology#nova-concepts">workspace</Term> `package.json` files.

## Summary

The `normalize-bundler` recipe handles unofficial <Term title="build tool" to="/docs/quickstart/terminology#builds-and-tooling">bundler</Term> convention fields in each <Term title="package" to="/docs/quickstart/terminology#nova-concepts">workspace</Term>'s `package.json`. It merges the deprecated `typings` field into `types`, and normalizes `module`, `sideEffects`, and `esnext` fields according to common bundler expectations.

## Usage

<Tabs groupId="install-state">
  <TabItem value="nova" label="nova (installed)" default>

    ```bash
    nova recipe package-json normalize-bundler [options]
    ```

  </TabItem>
  <TabItem value="npx" label="npx (no install)">

    ```bash
    npx --yes @cbnventures/nova@latest recipe package-json normalize-bundler [options]
    ```

  </TabItem>
</Tabs>

## Options

| Flag                 | Description                                               |
|----------------------|-----------------------------------------------------------|
| `-d, --dry-run`      | Preview changes without modifying files.                  |
| `-r, --replace-file` | Overwrite the original file instead of creating a backup. |
