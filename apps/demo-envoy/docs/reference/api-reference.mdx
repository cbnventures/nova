# API Reference

Envoy exposes a Spoke API for relay management, health monitoring, delivery inspection, and topic administration. All endpoints return JSON. Authentication uses the same Cipher mechanisms configured for relay endpoints.

## Base URL

```
http://localhost:8090/api
```

All endpoints below are relative to this base URL.

## Relay Endpoints

### Send a Message

```bash title="POST /api/relay/:topic"
curl -X POST http://localhost:8090/api/relay/infra-alerts \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ${RELAY_TOKEN}" \
  -d '{
    "title": "Manual alert",
    "body": "Triggered via API.",
    "priority": 3
  }'
```

```json title="Response — 200 OK"
{
  "id": "msg_a1b2c3d4",
  "status": "delivered",
  "relay": "manual",
  "latency_ms": 2.8
}
```

### Get Relay Status

```bash title="GET /api/relay/:name/status"
curl http://localhost:8090/api/relay/threadbare-pushes/status
```

```json title="Response — 200 OK"
{
  "relay": "threadbare-pushes",
  "active": true,
  "messages_today": 47,
  "delivery_rate": 1.0,
  "last_delivery": "2025-08-22T14:30:07Z",
  "dead_lettered": 0
}
```

## Topic Endpoints

### List Topics

```bash title="GET /api/topics"
curl http://localhost:8090/api/topics
```

```json title="Response — 200 OK"
{
  "topics": [
    { "name": "ci-builds", "relays": 2, "messages_today": 31 },
    { "name": "infra-alerts", "relays": 3, "messages_today": 12 },
    { "name": "oncall-urgent", "relays": 1, "messages_today": 4 },
    { "name": "monitoring", "relays": 2, "messages_today": 0 }
  ]
}
```

### Create Topic

```bash title="POST /api/topics"
curl -X POST http://localhost:8090/api/topics \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}" \
  -d '{ "name": "deploy-alerts", "retention": "30d" }'
```

```json title="Response — 201 Created"
{
  "topic": "deploy-alerts",
  "created": true,
  "retention": "30d"
}
```

### Update Topic

```bash title="PATCH /api/topics/:name"
curl -X PATCH http://localhost:8090/api/topics/deploy-alerts \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}" \
  -d '{ "retention": "90d" }'
```

### Delete Topic

```bash title="DELETE /api/topics/:name"
curl -X DELETE http://localhost:8090/api/topics/deploy-alerts \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

```json title="Response — 200 OK"
{
  "topic": "deploy-alerts",
  "deleted": true
}
```

### Retry a Dead-Lettered Message

```bash title="POST /api/deliveries/:id/retry"
curl -X POST http://localhost:8090/api/deliveries/msg_a9b8c7d6/retry \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

```json title="Response — 202 Accepted"
{
  "id": "msg_a9b8c7d6",
  "status": "pending",
  "message": "Message re-queued for delivery."
}
```

## Endpoint Summary

| Method   | Path                        | Description                     |
|----------|-----------------------------|---------------------------------|
| `POST`   | `/api/relay/:topic`         | Send a message to a topic.      |
| `GET`    | `/api/relay/:name/status`   | Get relay status and metrics.   |
| `GET`    | `/api/health`               | Health check.                   |
| `GET`    | `/api/health/metrics`       | Detailed platform metrics.      |
| `GET`    | `/api/topics`               | List all topics.                |
| `POST`   | `/api/topics`               | Create a new topic.             |
| `PATCH`  | `/api/topics/:name`         | Update topic configuration.     |
| `DELETE` | `/api/topics/:name`         | Delete a topic.                 |
| `GET`    | `/api/deliveries`           | List deliveries with filtering. |
| `POST`   | `/api/deliveries/:id/retry` | Retry a dead-lettered message.  |
| `GET`    | `/api/deliveries/:id`       | Inspect a single delivery.      |
