Normalize dependency fields and optionally **pin versions** in <Term title="package" to="/docs/quickstart/terminology#nova-concepts">workspace</Term> `package.json` files.

## Summary

The `normalize-dependencies` recipe handles all dependency-related fields in each <Term title="package" to="/docs/quickstart/terminology#nova-concepts">workspace</Term>'s `package.json`. It normalizes `dependencies`, `devDependencies`, `peerDependencies`, `optionalDependencies`, `bundleDependencies` (merging from the deprecated `bundledDependencies` form), and `overrides`. When version pinning settings are enabled, it strips range prefixes (e.g., `^`, `~`) from dependency versions.

## Settings

Settings are configured per workspace in `nova.config.json` under the `recipes` key:

| Setting                    | Type    | Description                                          |
|----------------------------|---------|------------------------------------------------------|
| `pinDependencyVersions`    | boolean | Strip range prefixes from `dependencies` versions    |
| `pinDevDependencyVersions` | boolean | Strip range prefixes from `devDependencies` versions |

## Options

| Flag                 | Description                                               |
|----------------------|-----------------------------------------------------------|
| `-d, --dry-run`      | Preview changes without modifying files.                  |
| `-r, --replace-file` | Overwrite the original file instead of creating a backup. |
