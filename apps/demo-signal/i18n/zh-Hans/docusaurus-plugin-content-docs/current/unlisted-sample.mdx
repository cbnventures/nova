# 未列出示例

当分析仪表盘的加载时间超过四秒时使用的速查表。按出现频率排序，最常见的三类元凶是：

- 没有按日期分区谓词就直接 join events 表（务必指定 `dt_event`——分区粒度是按天）。
- 在用户属性表上 `SELECT *`，而实际只需要两列（这张表有八十七列，绝大多数都很稀疏）。
- 漏斗查询里多余的 `DISTINCT`（events 表已经通过 `dedupe_key` 列做了天然去重）。

修掉其中任意一个，通常都能把仪表盘从四秒压到一秒以内。剩下的那一成慢查询则属于其他问题。
