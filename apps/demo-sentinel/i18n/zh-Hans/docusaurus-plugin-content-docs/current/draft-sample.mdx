# 草稿示例

关于探测间隔重做的工作笔记。当前默认值是 HTTP 探测 60 秒、TCP 探测 30 秒。

上个月我们遇到一次吵闹邻居（noisy-neighbor）事件——同一节点上的两个服务同时执行检查，宿主机因为完全来自我们自己制造的合成负载而触发了告警。

我们打算给每个节点加上正负 8 秒的抖动窗口，让探测自然错开，抖动种子从节点主机名派生以便在重启后保持稳定。我们不打算在它经过告警评估窗口的仿真之前就上线——漏报会比当前的状况糟糕得多。
