# Signal

Signal 是面向注意力经济的归因引擎。它追踪意图的流动——从初次接触到最终转化。每一条链接都是一只传感器，每一次重定向都是一个数据点。Signal 告诉你的不只是某个链接被点过，而是为什么被点，以及差一点发生了什么。

> "从一次点击到一次洞察之间的最短距离，是一条知道自己去过哪儿的链接。"

## Signal 做什么

大多数分析工具是在事后清点事件。Signal 则把仪器装到路径本身。每一条由 Signal 铸造出来的链接都携带嵌入式归因元数据——可穿越 URL 缩短服务、消息应用、隐私浏览模式以及跨设备的接力。当转化发生时，Signal 会重建完整旅程——每一个触点、每一次犹豫、每一个为转化贡献过的渠道。

| 工具            | 用途                         |
|---------------|----------------------------|
| **Beacon**    | 链接铸造引擎——带嵌入式归因元数据的品牌短链。    |
| **Prism**     | 多触点归因建模器——跨渠道追踪转化路径。       |
| **Pulse**     | 实时点击流——交互、地理位置、设备与来源的实时反馈。 |
| **Aperture**  | 营销活动快照——按时间点捕捉表现，并支持对比叠加。  |
| **Resonance** | 受众重叠探测器——找出哪些受众正在彼此交叉。     |
| **Flare**     | 二维码与深度链接生成器——以完整归因贯通线下到线上。 |

:::info
Signal 在链接层面运转。其他归因平台是把跟踪逻辑硬塞进页面里；Signal 则把它嵌入链接本身——所以早在页面加载之前，归因数据就已经存在。
:::

## 工作原理

Beacon 链接不只是一次重定向。它将营销活动、渠道、变体以及唯一的追踪标识一并编码进来。当链接被点击时，Pulse 会实时捕捉这次交互。当用户最终完成转化时，Prism 会沿着完整归因路径反向回溯，把功劳分配到每一个触点。

```
Beacon 铸造链接 → 用户点击 → Pulse 捕获事件
  → 用户从另一个渠道再次点击 → Pulse 再次捕获
    → 用户转化 → Prism 重建完整路径
      → Aperture 在转化时刻抓取营销活动状态
```

## 何时使用 Signal

- **营销活动归因** — 追踪哪些渠道、链接与触点驱动了转化。
- **受众智能** — 用 Resonance 探测营销活动与人群间的重叠。
- **实时监测** — 用 Pulse 看着点击流实时涌入。
- **线下到线上** — 用 Flare 把印刷、户外与活动场景接入数字归因。
- **暗流量回收** — 为那些剥离了 referrer 与 UTM 数据的点击找回归因。
