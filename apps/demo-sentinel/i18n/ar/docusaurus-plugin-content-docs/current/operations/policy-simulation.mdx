# محاكاة السياسة

Parapet هو بيئة محاكاة السياسات المعزولة في Sentinel. يختبر سياسات الوصول مقابل حركة بيانات تاريخية حقيقية دون تعديل أيّ حالة سياسة فعلية. وقبل أن تصل سياسة جديدة إلى الإنتاج، يُجيب Parapet عن السؤال، ما الذي سيتغيّر؟

## لماذا المحاكاة

تغييرات السياسة محفوفة بالمخاطر بطبيعتها. قاعدة تبدو صحيحة بمعزل عن غيرها قد تتفاعل مع السياسات القائمة بطرق غير متوقّعة، فتمنح وصولاً لم يكن موجوداً، أو تُلغي وصولاً عن حساب خدمة حرج. يُلغي Parapet هذه المخاطرة بإعادة تشغيل حركة بيانات أنفاق Filament المُسجَّلة عبر مجموعة السياسات المُقترحة، وإنتاج تقرير أثر دقيق.

:::warning المحاكاة ليست ضماناً
يُعيد Parapet تشغيل حركة بيانات تاريخية. يكشف كيف كانت السياسات المُقترحة ستُقيَّم مقابل أنماط الاتصال السابقة. ولا يستطيع التنبّؤ بأنماط حركة بيانات لم تحدث من قبل. اجمع دائماً بين محاكاة Parapet والنمذجة اليدويّة للتهديدات في بنى المناطق الجديدة أو أنماط الوصول المُستحدثة.
:::

## أوضاع المحاكاة

<Tabs>
<TabItem value="dryrun" label="التشغيل التجريبي" default>

يُقيّم وضع التشغيل التجريبي سياسة مسودّة مقابل الأسطول الحالي دون إعادة تشغيل حركة البيانات. يُظهر أيّ الجلسات النشطة حالياً ستنجح أو تُخفق تحت القواعد المُقترحة:

```bash title="محاكاة تشغيل تجريبي"
sentinel parapet dry-run \
  --policy draft:tightened-posture \
  --scope all-devices
```

<TabItem value="replay" label="إعادة تشغيل حركة البيانات">

تُعالج إعادة تشغيل حركة البيانات سجلّات أنفاق Filament التاريخية عبر مجموعة السياسات المُقترحة. تُظهر كيف كان سيُقيَّم كلّ اتصال خلال فترة زمنية:

```bash title="محاكاة إعادة تشغيل حركة البيانات"
sentinel parapet simulate \
  --policy-set production \
  --include draft:tightened-posture \
  --traffic-source production-replay \
  --duration 7d
```

## فرق السياسات

يستطيع Parapet مقارنة مجموعتَي سياسات جنباً إلى جنب، وإظهار فرق الوصول لكلّ مورد:

```bash title="فرق السياسات"
sentinel parapet diff \
  --baseline production \
  --proposed production+draft:tightened-posture
```

## ترقية محاكاة

بعد مراجعة نتائج المحاكاة، رقِّ سياسة المسودّة إلى الإنتاج:

```bash title="ترقية بعد المحاكاة"
sentinel parapet promote \
  --draft tightened-posture \
  --simulation-id sim_7f3a2b9c
```

تربط الترقية السياسة الجديدة بنتائج محاكاتها في Spyglass. يستطيع المدقّقون تتبّع كلّ سياسة إنتاج إلى المحاكاة التي صحّحتها.

:::note متطلّبات المحاكاة
يجب أن يمرّ كلّ تغيير سياسة يُعدّل شروط الوصول بمحاكاة Parapet قبل الترقية. يفرض ذلك مستوى التحكّم في Sentinel، تطبيق السياسة مباشرة دون محاكاة معطّل افتراضياً لموارد الإنتاج.
:::

## المحاكاة المؤتمتة في Conduit

ادمج Parapet في خطّ Conduit للتكامل المستمرّ لمحاكاة تغييرات السياسة في كلّ commit:

```text title="conduit.grain — policy simulation step"
pipeline "policy-deploy" {
  stage "simulate" {
    run = "sentinel parapet simulate --policy-set production --include $DRAFT_POLICY --duration 24h"
    fail_on = "anomalies > 0 || new_denials > 500"
  }

  stage "review" {
    type    = "manual"
    approvers = ["security-ops"]
  }

  stage "promote" {
    run = "sentinel parapet promote --draft $DRAFT_POLICY --simulation-id $SIM_ID"
  }
}
```

يُشغّل الخطّ إعادة تشغيل حركة بيانات لمدّة 24 ساعة، ويحجب الترقية إن اكتُشف شذوذ أو تجاوز عدد الرفوضات العتبة، ويتطلّب موافقة يدويّة من فريق العمليّات الأمنية، ثمّ يُرقّي السياسة تلقائياً.
