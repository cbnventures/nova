多年来，二维码一直背着名声不好的标签。它们是营销提案里的段子——一个被会上某人坚持放上去的元素，最后印在户外广告牌上。后来移动相机原生支持了二维码扫描，2020 到 2023 年间使用率涨了 400%。如今二维码确实在起作用，问题变成了：你的二维码策略有没有跟上。

我们分析了过去十二个月里经由 Beacon 路由的 800 万次扫码。数据讲了一个很清晰的故事：当二维码有上下文时，它转化得很好；当它通用泛化时，它转化得很糟。

## 有上下文的二维码 vs 通用二维码

通用二维码指向首页或一个落地页。带上下文的二维码会把营销活动元数据、位置信息与受众意图编码进 Beacon 重定向里。两者之间的表现差距相当显著：

| 二维码类型 | 扫码率  | 跳出率 | 转化率  |
|-------|------|-----|------|
| 通用    | 2.1% | 74% | 0.8% |
| 有上下文  | 2.3% | 41% | 3.6% |

扫码率几乎一样——二维码的物理摆放比它的编码内容更影响是否被扫。但跳出率与转化率却拉开了明显距离。带上下文的二维码把用户带到与其意图匹配的页面，这是关键差别。

## 用深度链接打通线下到线上

我们追踪到的最强二维码部署是深度链接——它直接把用户引导到移动应用中的某个具体界面，而不是一个网页：

```json title="二维码深度链接配置"
{
  "shortUrl": "https://go.signal.example/qr-menu-42",
  "destination": {
    "ios": "signal://menu/42?source=qr&loc=downtown",
    "android": "intent://menu/42#Intent;scheme=signal;package=com.signal.app;end",
    "fallback": "https://signal.example/menu/42"
  },
  "attribution": {
    "campaign": "restaurant-q2",
    "channel": "qr-print",
    "variant": "table-tent-downtown"
  }
}
```

深度链接把 iOS 与 Android 用户直接送进应用。未安装应用的用户则进入回退的网页 URL。Beacon 在重定向时根据 User-Agent 解析正确的目的地，并让归因元数据沿着所有路径同行。

## 印刷物的摆放位置很重要

我们在 200 家零售门店里按物理摆放位置追踪扫码率：

- **桌牌**：4.2% 扫码率（与视线平齐，闲置时间充足）
- **小票**：1.8% 扫码率（购买后，留有动机）
- **橱窗招贴**：0.6% 扫码率（路过流量，意图很低）
- **产品包装**：3.1% 扫码率（高参与度时刻）
- **活动胸牌**：5.7% 扫码率（被困住的受众，社交场景）

活动胸牌跑赢其它所有摆放方式。被困住的受众、社交场景下的动机，以及手机已经在手里的状态——这种组合制造出近乎理想的扫码条件。

## 给线下与线上之间架起桥

经由 Beacon 的每一次扫码都会留下完整归因记录。Prism 可以追踪到这样一条转化路径：在一场会议的胸牌上发生了一次实物二维码扫描，三天后在桌面浏览器上完成了注册。即便这两件事发生在不同设备、不同情境下，trace ID 也能把它们连起来。

这是大多数线下营销从未拥有过的那座桥：一种把物理触点与数字转化以确定方式连起来的纽带，不依赖 cookie、不靠指纹识别，也不靠猜。
