# 未列出示例

在任何会触发告警的事件中都使用这个格式。保持简短。两句话说明发生了什么。两句话说明对用户的影响是什么。一句话说明立即采取了什么修复措施。

接下来是真正重要的部分：我们的检测流水线里哪里失败了。我们在用户发现之前收到告警了吗？如果是，那是一个有用的告警还是嘈杂的告警？如果不是，缺失了哪些遥测数据，本来可以更早捕获这个问题？

除非有监管理由，否则不要费心写复杂的时间线。我们是想从中学到东西，而不是为审计师准备报告。
