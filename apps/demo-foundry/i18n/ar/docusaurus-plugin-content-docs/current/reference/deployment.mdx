# النشر

يشحن Foundry مساحات العمل إلى ثلاثة أهداف نشر: Arcline Cloud، وحاويات Vial، وتنسيق Trellis. يُهيَّأ كل هدف في البيان ويُنفَّذ بأمر واحد.

## أهداف النشر

<Tabs>
<TabItem value="arcline" label="Arcline Cloud" default>

Arcline منصة سحابية مُدارة. يُحزِّم Foundry خدمتك، ويُجهِّز البنية التحتية، ويُنشِرها دون خطوات يدوية.

```bash title="Deploy to Arcline"
foundry deploy --target arcline
```

```text title="Output"
→ Building api (service) for Arcline target
→ Bundling with Smelter (production mode)
→ Packaging deployment artifact (2.1 MB)
→ Provisioning Arcline instance: us-east-1
→ Deploying api@2.4.1 to arcline://platform-api.us-east-1
→ Health check passed (200 OK in 340ms)
→ Deployment complete. Live at https://api.platform.arcline.dev
```

</TabItem>
<TabItem value="vial" label="Vial Container">

تبني Vial صورة حاوية من حزمة خدمتك. تتضمّن الصورة المصدر المُترجَم، وتبعيات زمن التشغيل، وطبقة قاعدية بحدّها الأدنى.

```bash title="Build and push a Vial image"
foundry deploy --target vial
```

```text title="Output"
→ Building api (service) for Vial target
→ Generating Vialfile from manifest
→ Building image: foundry/platform-api:2.4.1
→ Image size: 18.4 MB (3 layers)
→ Pushing to registry.loom.dev/platform-api:2.4.1

```

</TabItem>
<TabItem value="trellis" label="Trellis Orchestration">

Trellis منصة تنسيق حاويات. يُولِّد Foundry بيان النشر ويُطبِّقه على عنقود Trellis الخاص بك.

## إعداد النشر

تتحكم كتلة `deploy` في بيانك بسلوك النشر:

```text title="project.grain — deploy configuration"
workspace "platform" {
  target = "arcline"

  deploy {
    region   = "us-east-1"
    replicas = 3
    scaling  = "auto"
    memory   = "512mb"
    timeout  = "30s"
  }

  packages {
    api { type = "service", depends = ["core"] }
  }
}
```

| التوجيه    | النوع     | الافتراضي     | الوصف                                           |
|------------|-----------|---------------|-------------------------------------------------|
| `region`   | `Text`    | `"us-east-1"` | منطقة النشر.                                    |
| `replicas` | `Numeric` | `1`           | عدد الحالات المُشغَّلة.                         |
| `scaling`  | `Text`    | `"manual"`    | وضع التوسّع: `manual` أو `auto` أو `scheduled`. |
| `memory`   | `Text`    | `"256mb"`     | تخصيص الذاكرة لكل حالة.                         |
| `timeout`  | `Text`    | `"30s"`       | مهلة الطلب لفحوصات السلامة.                     |
| `env`      | `Block`   | اختياري       | متغيّرات بيئة تُحقَن في وقت التشغيل.            |

## متغيّرات البيئة

ينبغي ألّا تعيش القيم الحسّاسة في البيان. استخدم كتلة `env` مع مراجع لمخزن أسرارك:

```text title="project.grain — environment variables"
deploy {
  env {
    DATABASE_URL = secret("platform/db-url")
    API_KEY      = secret("platform/api-key")
  }
}
```

تحلّ الدالة `secret()` القيم من مخزن أسرار منصة النشر في وقت النشر. لا يحتوي البيان أبدًا على بيانات اعتماد خام.

:::danger
تتطلب النشرات الإنتاجية العلم `--confirm`. تشغيل `foundry deploy --target arcline` دون `--confirm` سيُنفِّذ تشغيلًا جافًا ويطبع ما سيحدث دون النشر فعلًا. هذا إجراء سلامة لمنع الإصدارات الإنتاجية العرضية.
:::

:::note
العلم `--confirm` غير قابل للإعداد عبر البيان عمدًا. يجب أن يكون كل نشر إنتاجي قرارًا صريحًا وواعيًا في سطر الأوامر.
:::

## النشر متعدّد الحزم

حين تكون لمساحة عمل عدّة حزم خدمة، ينشرها Foundry بترتيب التبعيات:

```text title="Deploying a multi-service workspace"
$ foundry deploy --target arcline --confirm

→ Deployment plan:
  1. core (library) — skip (libraries are not deployed)
  2. auth (service) — deploy to arcline://auth.us-east-1
  3. api (service) — deploy to arcline://api.us-east-1 (depends on auth)
  4. gateway (service) — deploy to arcline://gateway.us-east-1 (depends on api, auth)

→ Deploying auth@2.4.1...  ✓ (4.2s)
→ Deploying api@2.4.1...   ✓ (3.8s)
→ Deploying gateway@2.4.1... ✓ (5.1s)

→ All services deployed. Total time: 13.1s
```

لا تُنشَر حزم المكتبات مباشرة أبدًا. تُحزَم في حزم الخدمات التي تعتمد عليها.

## التراجع

إذا فشل نشر أو لم يجتز فحص سلامة، يتراجع Foundry تلقائيًا:

```text title="Automatic rollback on failure"
→ Deploying api@2.5.0...
→ Health check failed (503 after 30s)
→ Rolling back to api@2.4.1
→ Rollback complete. Previous version restored.
```

للتراجع اليدوي، استخدم العلم `--rollback`:

```bash title="Manual rollback"
foundry deploy --target arcline --rollback api@2.4.1 --confirm
```
