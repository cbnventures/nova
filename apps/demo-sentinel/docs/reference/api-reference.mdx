# API Reference

The Sentinel Spoke API provides programmatic access to all platform operations — policy management, device queries, zone operations, and audit export. All endpoints require a Cipher bearer token for authentication.

## Authentication

Every request must include a Cipher bearer token in the `Authorization` header:

```bash title="Authenticated request"
curl -X GET https://sentinel.yourcompany.internal/spoke/v1/policies \
  -H "Authorization: Bearer $SENTINEL_API_TOKEN" \
  -H "Content-Type: application/json"
```

Generate tokens via the Sentinel dashboard or the CLI:

```bash title="Generate API token"
sentinel api tokens create \
  --name "ci-pipeline" \
  --scope "policies:read,policies:simulate,devices:read" \
  --ttl 90d
```

:::note Token Scopes
Tokens are scoped to specific operations. A token with `policies:read` cannot modify policies. A token with `devices:read` cannot enroll new devices. Always use the minimum scope required for your integration.
:::

## Policies

| Method   | Path                              | Description                        |
|----------|-----------------------------------|------------------------------------|
| `GET`    | `/spoke/v1/policies`              | List all policies                  |
| `GET`    | `/spoke/v1/policies/:id`          | Get a specific policy              |
| `POST`   | `/spoke/v1/policies`              | Create a new policy                |
| `PUT`    | `/spoke/v1/policies/:id`          | Update an existing policy          |
| `DELETE` | `/spoke/v1/policies/:id`          | Delete a policy                    |
| `POST`   | `/spoke/v1/policies/:id/evaluate` | Evaluate a policy against a device |
| `POST`   | `/spoke/v1/policies/simulate`     | Simulate a policy change (Parapet) |

### Create a Policy

```bash title="POST /spoke/v1/policies"
curl -X POST https://sentinel.yourcompany.internal/spoke/v1/policies \
  -H "Authorization: Bearer $SENTINEL_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "staging-access",
    "resource": "staging-services",
    "effect": "allow",
    "priority": 100,
    "conditions": {
      "device.posture": { "gte": 70 },
      "user.mfa": true,
      "user.role": ["engineer", "qa"]
    },
    "on_failure": {
      "action": "revoke",
      "notify": "security-ops"
    }
  }'
```

### Simulate a Policy Change

```bash title="POST /spoke/v1/policies/simulate"
curl -X POST https://sentinel.yourcompany.internal/spoke/v1/policies/simulate \
  -H "Authorization: Bearer $SENTINEL_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "proposed_changes": [{
      "policy_id": "pol_8a3f7b2c9d1e",
      "change": "update",
      "fields": {
        "conditions.device.posture.gte": 85
      }
    }],
    "scope": "all_devices",
    "mode": "dry-run"
  }'
```

## Devices

| Method | Path                             | Description                    |
|--------|----------------------------------|--------------------------------|
| `GET`  | `/spoke/v1/devices`              | List all enrolled devices      |
| `GET`  | `/spoke/v1/devices/:id`          | Get device details and posture |
| `POST` | `/spoke/v1/devices/:id/enroll`   | Enroll a device                |
| `POST` | `/spoke/v1/devices/:id/unenroll` | Remove a device from the fleet |
| `GET`  | `/spoke/v1/devices/:id/posture`  | Get current posture breakdown  |
| `GET`  | `/spoke/v1/devices/:id/history`  | Get posture history over time  |

```bash title="GET /spoke/v1/devices/:id/posture"
curl -X GET https://sentinel.yourcompany.internal/spoke/v1/devices/dev_a3b7c9d1/posture \
```

## Audit

### Query Audit Events

```bash title="GET /spoke/v1/audit/query"
curl -X GET "https://sentinel.yourcompany.internal/spoke/v1/audit/query?\
event_type=access-decision&\
result=revoke&\
period=2025-06-01..2025-06-30&\
resource=api-cluster-east" \
  -H "Authorization: Bearer $SENTINEL_API_TOKEN"
```

## Rate Limits

All Spoke API endpoints are rate-limited per token:

| Scope           | Limit              |
|-----------------|--------------------|
| Read endpoints  | 1,000 requests/min |
| Write endpoints | 100 requests/min   |
| Simulation      | 10 requests/hour   |
| Audit export    | 5 requests/hour    |

Rate limit headers are included in every response: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`.
