# مرجع API

يُقدّم Sentinel Spoke API وصولاً برمجياً إلى جميع عمليّات المنصّة، إدارة السياسات، واستعلامات الأجهزة، وعمليّات المناطق، وتصدير التدقيق. تتطلّب جميع نقاط النهاية رمز bearer من Cipher للمصادقة.

## المصادقة

يجب أن يتضمّن كلّ طلب رمز bearer من Cipher في رأس `Authorization`:

```bash title="طلب مُصادَق"
curl -X GET https://sentinel.yourcompany.internal/spoke/v1/policies \
  -H "Authorization: Bearer $SENTINEL_API_TOKEN" \
  -H "Content-Type: application/json"
```

ولّد الرموز عبر لوحة تحكّم Sentinel أو واجهة سطر الأوامر:

```bash title="توليد رمز API"
sentinel api tokens create \
  --name "ci-pipeline" \
  --scope "policies:read,policies:simulate,devices:read" \
  --ttl 90d
```

:::note نطاقات الرموز
تخضع الرموز لنطاقات محدّدة من العمليّات. الرمز ذو `policies:read` لا يستطيع تعديل السياسات. والرمز ذو `devices:read` لا يستطيع تسجيل أجهزة جديدة. استخدم دائماً الحدّ الأدنى من النطاقات المطلوب لتكاملك.
:::

## السياسات

| الطريقة  | المسار                            | الوصف                        |
|----------|-----------------------------------|------------------------------|
| `GET`    | `/spoke/v1/policies`              | عرض جميع السياسات            |
| `GET`    | `/spoke/v1/policies/:id`          | الحصول على سياسة محدّدة      |
| `POST`   | `/spoke/v1/policies`              | إنشاء سياسة جديدة            |
| `PUT`    | `/spoke/v1/policies/:id`          | تحديث سياسة موجودة           |
| `DELETE` | `/spoke/v1/policies/:id`          | حذف سياسة                    |
| `POST`   | `/spoke/v1/policies/:id/evaluate` | تقييم سياسة مقابل جهاز       |
| `POST`   | `/spoke/v1/policies/simulate`     | محاكاة تغيير سياسة (Parapet) |

### إنشاء سياسة

```bash title="POST /spoke/v1/policies"
curl -X POST https://sentinel.yourcompany.internal/spoke/v1/policies \
  -H "Authorization: Bearer $SENTINEL_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "staging-access",
    "resource": "staging-services",
    "effect": "allow",
    "priority": 100,
    "conditions": {
      "device.posture": { "gte": 70 },
      "user.mfa": true,
      "user.role": ["engineer", "qa"]
    },
    "on_failure": {
      "action": "revoke",
      "notify": "security-ops"
    }
  }'
```

### محاكاة تغيير سياسة

```bash title="POST /spoke/v1/policies/simulate"
curl -X POST https://sentinel.yourcompany.internal/spoke/v1/policies/simulate \
  -H "Authorization: Bearer $SENTINEL_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "proposed_changes": [{
      "policy_id": "pol_8a3f7b2c9d1e",
      "change": "update",
      "fields": {
        "conditions.device.posture.gte": 85
      }
    }],
    "scope": "all_devices",
    "mode": "dry-run"
  }'
```

## الأجهزة

| الطريقة | المسار                           | الوصف                                   |
|---------|----------------------------------|-----------------------------------------|
| `GET`   | `/spoke/v1/devices`              | عرض جميع الأجهزة المسجّلة               |
| `GET`   | `/spoke/v1/devices/:id`          | الحصول على تفاصيل الجهاز ووضعه الأمني   |
| `POST`  | `/spoke/v1/devices/:id/enroll`   | تسجيل جهاز                              |
| `POST`  | `/spoke/v1/devices/:id/unenroll` | إزالة جهاز من الأسطول                   |
| `GET`   | `/spoke/v1/devices/:id/posture`  | الحصول على تفصيل الوضع الأمني الحالي    |
| `GET`   | `/spoke/v1/devices/:id/history`  | الحصول على تاريخ الوضع الأمني عبر الزمن |

## المناطق

| الطريقة | المسار                          | الوصف                                  |
|---------|---------------------------------|----------------------------------------|
| `GET`   | `/spoke/v1/zones`               | عرض جميع مناطق Rampart                 |
| `GET`   | `/spoke/v1/zones/:id`           | الحصول على تفاصيل المنطقة وأعباء العمل |
| `POST`  | `/spoke/v1/zones`               | إنشاء منطقة جديدة                      |
| `PUT`   | `/spoke/v1/zones/:id`           | تحديث تهيئة المنطقة                    |
| `GET`   | `/spoke/v1/zones/:id/crossings` | عرض قواعد العبور للمنطقة               |
| `POST`  | `/spoke/v1/zones/crossings`     | إنشاء قاعدة عبور منطقة                 |

## التدقيق

| الطريقة | المسار                         | الوصف                       |
|---------|--------------------------------|-----------------------------|
| `GET`   | `/spoke/v1/audit/query`        | الاستعلام عن أحداث Spyglass |
| `GET`   | `/spoke/v1/audit/sessions/:id` | إعادة بناء جلسة محدّدة      |
| `POST`  | `/spoke/v1/audit/export`       | تصدير حزمة أدلّة الامتثال   |
| `GET`   | `/spoke/v1/audit/export/:id`   | تنزيل تصدير مكتمل           |

### الاستعلام عن أحداث التدقيق

```bash title="GET /spoke/v1/audit/query"
curl -X GET "https://sentinel.yourcompany.internal/spoke/v1/audit/query?\
event_type=access-decision&\
result=revoke&\
period=2025-06-01..2025-06-30&\
resource=api-cluster-east" \
  -H "Authorization: Bearer $SENTINEL_API_TOKEN"
```

## حدود المعدّل

جميع نقاط نهاية Spoke API محدودة المعدّل لكلّ رمز:

| النطاق             | الحدّ           |
|--------------------|-----------------|
| نقاط نهاية القراءة | 1,000 طلب/دقيقة |
| نقاط نهاية الكتابة | 100 طلب/دقيقة   |
| المحاكاة           | 10 طلبات/ساعة   |
| تصدير التدقيق      | 5 طلبات/ساعة    |

تُضمَّن رؤوس حدود المعدّل في كلّ استجابة، `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`.
