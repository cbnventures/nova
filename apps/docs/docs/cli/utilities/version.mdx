Generate Markdown-ready tables of version information for your OS, developer tools, package managers, browsers, and runtimes in a single run.

## Summary

The `version` command standardizes results for quick copy and paste into issues or docs. It collects version data for developer tools, operating systems, major browsers, and language runtimes.

:::tip Filing a GitHub Issue?
Paste the tables as is. The output is **Markdown-ready** for quick reports.
:::

## Why Use This Command?

- Check multiple categories at once (Node.js, package managers, OS, browsers, interpreters).
- Produce uniform tables so no details are missed.
- Work across macOS, Windows, and Linux with platform-specific detection.
- Give maintainers a clear system profile when you report a bug.

## Use Cases

- **Bug reporting** — Attach system information to GitHub issues without manually running multiple commands.
- **Team collaboration** — Share consistent system snapshots with teammates for debugging or onboarding.
- **CI/CD pipelines** — Record system information in build logs or artifacts for reproducibility.
- **System verification** — Confirm that required tools are installed and up to date before starting development.

## Requirements

- **Supported systems** — macOS, Windows, and Linux are fully tested.
- **Permissions** — Runs under standard user accounts; no elevated privileges required.
- **Network access** — Not needed; all checks are performed locally.

## Usage

You can run this command in two ways:

<Tabs groupId="install-state">
  <TabItem value="nova" label="nova (installed)" default>

    # Original
    nova utility version

    # Shorthand
    nova util ver
    ```

    **Note:** If Nova is installed **only locally** (inside a project), the command will work **only within that project's directory**. See the [npm docs](https://docs.npmjs.com/cli/configuring-npm/folders) for details.

  </TabItem>

    # Shorthand
    npx --yes @cbnventures/nova@latest util ver
    ```

    **Note:** If you prefer the shorter `nova` command, see the [Setup and Configure](/docs/quickstart/setup) guide for installation instructions.

  </TabItem>
</Tabs>

## Options

| Flag                | Description                                        |
|---------------------|----------------------------------------------------|
| `-a, --all`         | Show all available versions.                       |
| `-b, --browser`     | Show web browser versions.                         |
| `-e, --env`         | Show environment manager versions.                 |
| `-i, --interpreter` | Show interpreter / runtime versions.               |
| `-n, --node`        | Show Node.js and related package manager versions. |
| `-o, --os`          | Show operating system details.                     |

## Categories

Each option enables checks for a specific category of tools or system information.

### Node.js + Tools

- Node.js
- Node Package Manager (npm)
- Yarn
- Performant Node Package Manager (pnpm)
- Bun

<details>
  <summary>How Node.js tool version detection works</summary>
  <ul>
    <li><strong>Node.js</strong>: Read from <code>{'node --version'}</code>.</li>
    <li><strong>npm</strong>: Read from <code>{'npm --version'}</code>.</li>
    <li><strong>Yarn</strong>: Read from <code>{'yarn --version'}</code>.</li>
    <li><strong>pnpm</strong>: Read from <code>{'pnpm --version'}</code>.</li>
    <li><strong>Bun</strong>: Read from <code>{'bun --version'}</code>.</li>
  </ul>
</details>

### Environment Managers

- Node Version Manager (nvm)
- NVM for Windows
- Volta

**Note**: On Windows, **NVM for Windows** is identified separately from **Node Version Manager**, since the original `nvm` only works in POSIX-compliant shells (macOS / Linux).

<details>
  <summary>How environment manager version detection works</summary>
  <ul>
    <li><strong>nvm (POSIX)</strong>: Read from <code>{'nvm --version'}</code>.</li>
    <li><strong>nvm (Windows)</strong>: Read from <code>{'nvm --version'}</code>.</li>
    <li><strong>Volta</strong>: Read from <code>{'volta --version'}</code>.</li>
  </ul>
</details>

### Operating System

- OS Name (e.g., macOS, Windows, Linux distros)
- OS Version (if available)
- OS Architecture (e.g., x64, arm64)
- OS Build number (if available)
- OS Kernel

<details>
  <summary>How OS version detection works</summary>
  <ul>
    <li><strong>macOS</strong>: Read from the <code>{'sw_vers'}</code> command.</li>
    <li><strong>Windows</strong>: Read from the <code>{'CurrentVersion'}</code> key inside Windows Registry.</li>
    <li><strong>Linux</strong>: Read from the <code>{'/etc/os-release'}</code> file.</li>
  </ul>
</details>

:::note Legend
- ✅ — Detected on this platform
- **N/A** — Not available on this operating system
- ⚠️ — Intentionally excluded
:::

<details>
  <summary>How browser version detection works</summary>
  <ul>
    <li><strong>macOS</strong>: Read from <code>{'*.app/Contents/Info.plist'}</code> using <code>{'CFBundleShortVersionString'}</code>.</li>
    <li><strong>Windows</strong>: Read from the executable's file metadata <code>{'VersionInfo.ProductVersion'}</code> via Windows Registry App Paths.</li>
    <li><strong>Linux</strong>: Parsed from the output of running the binary on <code>{'PATH'}</code> with <code>{'--version'}</code>.</li>
  </ul>
</details>

### Interpreters / Runtimes

- Java (version, distribution, build info)
- Rust (version, build hash, build date, source)

<details>
  <summary>How interpreter version detection works</summary>
  <ul>
    <li><strong>Java</strong>: Parsed from <code>{'java --version'}</code>.</li>
    <li><strong>Rust</strong>: Parsed from <code>{'rustc --version'}</code>.</li>
  </ul>
</details>

## Troubleshooting

- **Empty categories** — If a tool is not installed on your system, it will not appear in the output table.
- **Permission issues** — Elevated privileges are **not** required. If errors occur, check your PATH instead of using `sudo`.
- **Browser detection** — Certain browsers (e.g., LibreWolf) are platform-specific or may not be supported. Refer to [the table](/docs/cli/utilities/version#web-browsers) above to confirm availability.

## Security / Privacy

- Operates **entirely on the local machine**; no data is sent externally.
- Collects only tool names, versions, and OS metadata (e.g., architecture, build, kernel).
- Does **not** expose sensitive files, credentials, or environment variables.
