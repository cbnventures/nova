一台把一切都叫作入侵者的动作传感器，并不是一套安防系统——它只是一个停不下来的门铃。我们引入了置信带，好让 lantern 能分辨猫与门。

大多数动作都很小。凌晨三点的猫很小，穿过纱网门廊的风也很小。Lantern 现在会等到第二种特征出现之后，才悄悄打出一面旗。

## 第一个吵闹的月份

第一个月很吵。每次猫经过过道传感器，厨房的平板都会"叮"地响一下；每次飞蛾停在门廊镜片上，仪表盘都会闪一下。我们看着家里人开始对这些提示视而不见——这是任何报警系统都难以幸免的失败模式。一个总在喊"狼来了"的警报，只是一只没人对时的钟。

## 特征与阈值

于是我们坐下来翻看传感器历史，开始为所见到的现象起名。猫是一组贴近地面的低位动作——离地约三十厘米，沿着过道地毯缓慢漂移。人是一根柱子——胸高、更快、带有步伐节奏。风是一阵抖动——四分之一秒的高幅度，然后是长长的沉寂。洗碗机是厨房传感器上的一段嗡鸣，多半在晚上 9:14 开始，持续九十分钟。

每一种模式都得到一条置信带。不是是或否——而是一个 0 到 1 之间的数字，每个分区、每种模式都有不同的阈值。

```yaml title="lantern.events.yml"
confidence:
  hallway:
    motion-hallway:
      cat:        { signature: low-floor-drift,  threshold: 0.85 }
      person:     { signature: column-rhythm,    threshold: 0.65 }
      dishwasher: { signature: harmonic-9-14pm,  threshold: 0.40 }
  porch:
    motion-front-door:
      person:     { signature: column-rhythm,    threshold: 0.55 }
      wind:       { signature: high-flutter,     threshold: 0.95 }
      cat:        { signature: low-floor-drift,  threshold: 0.90 }
```

阈值很关键。门廊宽容——它认识风、认识猫，过去什么都见过。过道更严厉，但仅在夜里。凌晨三点的人需要一个低阈值；下午三点的人则是家里人，不需要阈值。

## 信任作为副产物

我们没料到的是，当系统不再说谎之后，家里人对它的信任有多深。三月里那块厨房平板有好几周都没出过声。一个周六它响了，所有人都抬起头——猫站在台面上，那正是我们想要的提示。

置信带主要是谦逊。它是 lantern 在承认自己尚不确定，并在开口之前等待第二种特征。过于笃定的代价，是被人忽略；略带迟疑的代价，是被人信任。我们选了后者。

## 自己启动的洗碗机

三月里有一件小事让我们吃了一惊。一个周六上午 9:13，洗碗机的特征带触发了，几乎正点，可没有洗碗机在运转。厨房平板没有响——这个数值在它的提示阈值之下——但那一行还是写进了历史。我们正午时分读到那一行，走下楼去看。洗碗机因为一个卡住的按钮，自己启动了一次清洗循环。Lantern 没有提醒我们，因为这种模式太熟悉；它只是把这一行写了下来。等我们找到那一行时，我们其实已经知道在看的是什么了——这正是这条置信带处在那个位置的全部意义。

```bash
$ lantern history --zone kitchen --since "this morning" --confidence
09:13   motion-kitchen   harmonic-9-14pm   0.42   below threshold (0.40)   logged
09:14   motion-kitchen   harmonic-9-14pm   0.51   below threshold           logged
...
```

洗碗机的置信带刻意被设得很低。它被允许在不被宣告的情况下存在。它运转时，那一行会被写下；它在错误时间运转时，那一行同样会被写下。家里人想读的时候才读。厨房从不为洗碗机提高声音，从不。这同样是一种谦逊。
