# Audit & Forensics

Spyglass is Sentinel's audit and forensics engine. It records every access decision, every policy evaluation, every zone crossing event, and every Filament tunnel connection — immutably, for seven years. The purpose is not just retention. It is reconstruction. Any session, any incident, any access pattern can be replayed step by step from the log data alone.

## What Spyglass Records

Every event in the Sentinel trust loop is captured by Spyglass:

| Event Type        | Source     | Data Captured                                                     |
|-------------------|------------|-------------------------------------------------------------------|
| Policy evaluation | Watchtower | User, device, resource, conditions, score, result, timestamp      |
| Access decision   | Drawbridge | Grant/narrow/revoke, policy ID, tunnel ID, duration               |
| Zone crossing     | Rampart    | Source zone, target zone, crossing rule, policy evaluation result |
| Posture check     | Garrison   | Device ID, all check categories, scores, pass/fail per category   |
| Tunnel event      | Filament   | Tunnel ID, endpoints, duration, bytes transferred, close reason   |
| Emergency access  | Admin      | User, justification, scope, duration, reviewer                    |

All events are cryptographically signed and stored in append-only logs. Records cannot be modified or deleted — even by administrators.

:::info Immutable by Design
Spyglass logs are written to append-only storage with cryptographic chaining. Each record includes a hash of the previous record, forming a tamper-evident chain. Any modification to a historical record breaks the chain and triggers an integrity alert.
:::

## Session Reconstruction

Spyglass can reconstruct an entire access session from its component events. This is the primary forensics tool for incident investigation:

```bash title="Reconstruct a session"
sentinel spyglass reconstruct \
  --user j.chen \
  --device dev_a3b7c9d1 \
  --period "2025-06-15T09:00:00..2025-06-15T17:00:00"
```

## Audit Queries

Spyglass supports structured queries for searching across all event types:

```bash title="Query by resource and time range"
sentinel spyglass query \
  --resource api-cluster-east \
  --event-type access-decision \
  --result revoke \
  --period "2025-06-01..2025-06-30"
```


<summary>Full query parameter reference</summary>

| Parameter      | Type   | Description                                                                              |
|----------------|--------|------------------------------------------------------------------------------------------|
| `--resource`   | String | Filter by resource name (supports wildcards)                                             |
| `--user`       | String | Filter by user identity                                                                  |
| `--device`     | String | Filter by device ID                                                                      |
| `--event-type` | Enum   | `policy-evaluation`, `access-decision`, `zone-crossing`, `posture-check`, `tunnel-event` |
| `--result`     | Enum   | `pass`, `fail`, `grant`, `narrow`, `revoke`                                              |
| `--period`     | Range  | ISO 8601 date range (`start..end`)                                                       |
| `--zone`       | String | Filter by Rampart zone name                                                              |
| `--policy`     | String | Filter by policy name                                                                    |
| `--sort`       | Enum   | `time`, `user`, `resource`, `score`                                                      |
| `--format`     | Enum   | `table`, `json`, `csv`                                                                   |

</Details>

## Compliance Export

Spyglass generates compliance evidence packages for standard frameworks:

```bash title="Export compliance evidence"
sentinel spyglass export --framework soc2 --period 2025-Q2
sentinel spyglass export --framework hipaa --period 2025-Q2
sentinel spyglass export --framework iso27001 --period 2025-Q2
```

Each export produces a structured evidence package containing policy evaluation summaries, access decision statistics, posture check distributions, and zone crossing logs — formatted for the specific compliance framework.
