我们职业生涯前半段都在建墙：防火墙、外围防御、网关设备——全都建立在同一个假设之上：网络有"里面"和"外面"，里面是安全的。

{/* truncate */}

这个假设一直是错的。我们只是当时没意识到。

## 那条从未存在过的边界

> "边界早在多年前就已消解。整个行业花了十年假装它还在。"

2015 年的企业网络拓扑很干净：员工从办公室连入，服务器在数据中心，防火墙坐在两者之间决定谁能进。你在里面，就被信任；你在外面，就不被信任。

后来远程办公出现了，承包商出现了，多云出现了。网络不再有"里面"，而防火墙还守着那条早已不存在的边界。

## 一条传统防火墙规则长什么样

下面是一条传统的边界规则。它基于源 IP 授予访问——一个关于你在哪里的静态事实，而不是关于你是谁、或你在做什么：

```text title="传统防火墙规则"
ALLOW src=10.0.0.0/8 dst=api-cluster port=443
DENY  src=* dst=api-cluster port=443
```

如果你在企业网内，你就进得来；如果你的凭证在该地址段内被攻陷，攻击者也进得来。防火墙分不出区别。

## 一条 Sentinel 策略长什么样

下面是同一次访问决策，以 Sentinel 信任策略的形式表达。它持续评估，而不是只评估一次：

```text title="trust-policy.grain"
policy "api-access" {
  resource = "api-cluster"
  effect   = "allow"

  conditions {
    device.posture   >= 80
    user.mfa         = true
    network.location = ["office", "vpn"]
    behavior.score   >= 70
  }

  on_failure {
    action = "revoke"
    notify = "security-ops"
  }
}
```

这条策略不在乎你在哪里。它在乎你的设备是否健康、你是否使用了第二因素认证、你的行为是否与历史一致。它每 90 秒检查一次。

:::danger 遗留的边界假设
信任一个 IP 段的防火墙规则会信任该段内的每一台设备——无论是否被攻陷。Sentinel 不把信任授予网络，它把信任授予"人、设备、上下文"的组合，并在其中任意一项发生变化时立刻撤销。
:::

## 我们为什么做 Sentinel

最初的观察很简单：每一次 VPN 都是一次信任假设。你认证一次，之后整个会话期间，网络都假设你还是那个人、用着同一台设备、抱着同样的意图。这个假设本身就是漏洞。

我们做 Sentinel 就是为了消除它。不是建一堵更好的墙，而是用持续评估取代墙：Watchtower 每 90 秒检查态势，Drawbridge 实时调整访问，Garrison 在每个端点上执行合规，Rampart 把网络分段，让一次成功的攻陷也无法扩散。

> "访问不是一扇打开后就敞开的门，而是一场永不停止的对话。"

这就是原则。网络不被信任，用户不被信任，设备不被信任。信任由证据计算而来、被持续重新评估，并在证据改变的那一刻被撤销。

## 改变是什么

在我们内部部署 Sentinel 之后：

- **凭证盗用事件归零。** 被盗的密码在没有合规设备与通过的态势检查下是无用的。
- **横向移动变得不可能。** Rampart 区域意味着被攻陷的服务账号无法触达其所在段之外的任何东西。
- **审计准备从数周缩短到数小时。** Spyglass 维护一份 7 年不可篡改日志，合规证据永远是最新的。

网络并不安全。它从来都不安全。我们停止假装，开始验证。
