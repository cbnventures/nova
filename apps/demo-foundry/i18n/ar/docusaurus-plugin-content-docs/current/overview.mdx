# البدء

Foundry هو مُترجم لنوايا المطوّر. تصف الشكل الذي ينبغي أن تكون عليه مساحة عملك في ملف بيان `.grain` وحيد، ويطرق Foundry بيئة التطوير بأكملها — بنية الدلائل، ورسم التبعيات، وخط إنتاج البناء، وقواعد التدقيق، وهياكل الاختبار.

لا ملفات إعداد، ولا أسلاك يدوية. بيان واحد يدخل، وبيئة كاملة تخرج.

## ما الذي يفعله Foundry

يقرأ Foundry ملف بيان `.grain` ويُنتج كل ما تحتاجه مساحة عمل متعددة الحزم للبناء والتدقيق والاختبار والنشر. بدلًا من صيانة عشرات الملفات الإعدادية عبر الحزم، تصون ملفًا واحدًا يُعلن نواياك.

| الأداة       | الغرض                                                           |
|--------------|-----------------------------------------------------------------|
| **Anvil**    | واجهة الأوامر الأساسية — تقرأ بيان Blueprint وتطرق مساحة العمل. |
| **Tongs**    | محلِّل ومُصوِّر لرسم التبعيات.                                  |
| **Quench**   | ذاكرة بناء تزايدية — لا تُعيد بناء سوى ما تغيّر.                |
| **Bellows**  | مُشغِّل مهام متوازية عبر مساحات العمل.                          |
| **Crucible** | مُولِّد منصّة اختبار — يكتب هياكل اختبار من تواقيع الأنواع.     |
| **Slag**     | كاشف للشيفرة الميتة والاعتماديات غير المستخدمة.                 |

## البدء السريع

ثبِّت Foundry عالميًا عبر Spark:

```bash title="Install Foundry"
spark install foundry
```

أنشئ بيان `.grain` بسيطًا في جذر مشروعك:

```text title="project.grain"
workspace "my-app" {
  lang   = "alloy"
  warden = ["strict"]

  packages {
    core { type = "library" }
    api  { type = "service", depends = ["core"] }
  }
}
```

اطرق مساحة العمل:

```bash title="Run your first forge"
foundry ignite
```

يقرأ Foundry البيان، ويحلّ رسم التبعيات، ويُولِّد قواعد Warden، ويُهيِّكل ملفات الاختبار، ويكتب كل أثر إعدادي تحتاجه حزمك.

:::tip
شغِّل `foundry ignite --dry-run` لمعاينة ما سيُولِّده Foundry دون كتابة أي ملفات على القرص. هذه أسرع وسيلة للتحقق من البيان قبل الإيداع.
:::
