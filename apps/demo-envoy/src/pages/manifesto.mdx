# Manifesto

Envoy exists because integration is a translation problem, not an authority problem. We do not believe both sides of a system should have to agree on a format before they can talk. The system that arrived first should not have to bend. The system that arrived last should not have to bend either. The translator sits in the middle and absorbs the difference, so neither side knows there is an interpreter in the room.

## The Posture of Translation

The translator does not choose sides. The translator removes the need to choose. When a request leaves the sending system, it is shaped for the sender. When it arrives at the receiving system, it is shaped for the receiver. Both sides experience the exchange as native, because the translator did the only work that had to be done — the shaping in between.

## What We Refuse

We refuse the framework that demands both sides convert before they can talk. We refuse the contract that treats one party's format as the canonical one and another's as the legacy. We refuse the assumption that interoperability requires a steering committee.

## How We Build

We build adapters first and opinions later. This is also how good documentation works — we build for the systems that are already running, not for the systems we wish someone had built. An adapter that handles a real format poorly is more useful than a clean contract that handles no real format at all.

### Adapters before opinions

The format predates the standard. The system predates the spec. We meet what is, not what was supposed to be.
