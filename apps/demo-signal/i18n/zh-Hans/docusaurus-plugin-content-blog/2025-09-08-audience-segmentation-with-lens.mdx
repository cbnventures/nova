人口属性告诉你一个人是谁。行为告诉你他想要什么。我们围绕这条分界做了 Lens。传统的受众细分按年龄、地点与职位归类用户；Lens 则按他们点过什么、回访多频繁，以及哪些内容把他们推近了转化来细分。

结果差距并不微弱。在我们度量过的账户里，行为型细分在转化率上比人口属性型细分平均高出 2.4 倍。

## 人口属性 vs 行为型细分

我们在 30 个同时拥有人口属性数据（来自 CRM 补全）与行为数据（来自 Beacon 点击历史）的账户上做了一组对照实验。每个账户都把它们接下来的一次营销活动拆成两种受众策略：

| 策略   | 平均 CTR | 平均转化率 | 单次获取成本 |
|------|--------|-------|--------|
| 人口属性 | 1.8%   | 1.2%  | $42.00 |
| 行为型  | 2.6%   | 2.9%  | $17.50 |

行为型定向同时带来了更高的点击率、超过两倍的转化率，以及不到一半的获取成本。原因不复杂：过去行为对未来行为的预测力，远胜过一个职位名称。

## Lens 如何构建细分

Lens 分析 Beacon 点击数据来识别行为聚类。用户对每一条 Beacon 链接的交互，都会进入他的行为画像：

- **内容亲和度**：他们访问了哪些文档页面、博文与产品页。
- **参与速度**：从第一次点击到再次访问之间的间隔。
- **渠道偏好**：他们更常通过邮件、社交、搜索还是直接到达。
- **距转化的距离**：通常在转化前还需要经过几个触点。

Lens 基于这些信号将用户分组：

```json title="Lens 细分定义"
{
  "segment": "high-intent-evaluators",
  "criteria": {
    "contentAffinity": ["pricing", "comparison", "case-study"],
    "engagementVelocity": { "min": 3, "period": "7d" },
    "conversionProximity": { "touchpointsRemaining": { "max": 2 } }
  },
  "size": 4280,
  "conversionRate": 8.7
}
```

这一细分包含那些访问过定价与对比页、过去一周回访至少三次、并且在统计意义上距离转化只剩不到两个触点的用户。他们不是按行业或公司规模定义的——而是按他们做过什么定义的。

## 自我更新的细分

静态细分会过期。一份在一月份建好的"企业潜客"列表，到三月就已经走样。Lens 的细分是动态的——随着新的行为数据从 Beacon 持续流入，成员归属会被持续重算。

上一周还在 `early-explorers` 细分里的用户，本周如果他的点击模式变了，就可能挪到 `high-intent-evaluators`。细分的边界由行为定义，而行为是会变的。细分也因此自适应。

## 把 Lens 接到 Flare

Lens 细分会直接喂给 Flare 的营销活动定向。当你在 Flare 中新建营销活动时，你可以把 Lens 细分作为目标——可以替代人口属性列表，也可以与之并用。这样营销活动触达的，是那些"行为暗示已经准备好接收这条信息"的用户，而不是那些"刚好符合某种人口属性的人"。

这种集成把闭环关上了：Beacon 捕获行为、Lens 把它细分、Flare 据此定向、Prism 度量结果，下一次 Lens 细分又把新数据吸进来。每一轮循环都让细分更锋利、让营销活动更高效。

## 从你现有的东西开始

Lens 不要求接 CRM，也不需要数据仓库。它从 Beacon 点击数据本身就能构建细分。如果你已经在用 Beacon 链接，Lens 需要的行为数据其实你早就有了。细分就在那儿等着——你只需要去看。
