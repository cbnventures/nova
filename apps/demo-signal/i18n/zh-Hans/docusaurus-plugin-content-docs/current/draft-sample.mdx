# 草稿示例

转化归因作业一直在抛出包含了上线周峰值的七日滚动均值，这意味着下游所有内容都在与一个本应排除在基线之外的近期峰值进行比较。

可选的方案要么是补一段合成基线，用线性方式把上线前与上线后两个时间窗连起来；要么干脆把上线周从滚动窗口中剔除。笔记本里的草图显示，剔除方式更干净，向产品团队解释也更省事。

我们会写一份一页纸的说明并先在内部传阅，再去动那个 cron。在 Eng-Data 团队点头之前，不会提交此项。
