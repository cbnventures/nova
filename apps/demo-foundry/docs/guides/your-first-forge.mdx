# Your First Forge

This guide walks you through creating a workspace from scratch using Foundry. You will write a `.grain` manifest, run your first forge, and inspect the output.

## Step 1: Create the Project Directory

Start with an empty directory:

```bash
mkdir my-project && cd my-project
```

## Step 2: Write the Manifest

Create a `project.grain` file at the root. This is the only file Foundry needs.

```text title="project.grain"
# A minimal workspace with two packages.
workspace "my-project" {
  lang    = "alloy"
  target  = "arcline"
  warden  = ["strict", "conventions"]
  // highlight-start
  crucible = auto
  // highlight-end

  packages {
    core { type = "library" }
    api  { type = "service", depends = ["core"] }
  }
}
```

Line 6 is the key directive. Setting `crucible = auto` tells Foundry to generate test scaffolds from your type signatures automatically. You can also set it to `manual` if you prefer to write tests yourself.

## Step 3: Run the Forge

Execute the `foundry ignite` command:

```bash title="Forge the workspace"
foundry ignite
```

Foundry reads your manifest, resolves the dependency graph, generates configuration artifacts, and scaffolds every package.

```text title="Terminal output"
→ Reading project.grain
→ Resolved workspace "my-project" with 2 packages
→ Dependency graph: core → api
→ Forging package: core (library)
    ✓ Generated blueprint.grain
    ✓ Generated .wardenrc.grain (strict + conventions)
    ✓ Generated alloy.config.grain
    ✓ Crucible scaffolded 4 test files from 4 type exports
→ Forging package: api (service)
    ✓ Generated blueprint.grain
    ✓ Generated .wardenrc.grain (strict + conventions)
    ✓ Generated alloy.config.grain
    ✓ Generated spoke.config.grain (service target)
    ✓ Crucible scaffolded 6 test files from 6 type exports
→ Done in 3.8s
```

## Step 4: Inspect the Output

After forging, your directory looks like this:

```text title="Generated workspace structure"
my-project/
  project.grain
  packages/
    core/
      blueprint.grain
      .wardenrc.grain
      alloy.config.grain
      src/
      tests/
        format.assay.al
        validate.assay.al
        parse.assay.al
        transform.assay.al
    api/
      blueprint.grain
      .wardenrc.grain
      alloy.config.grain
      spoke.config.grain
      src/
      tests/
        routes.assay.al
        handlers.assay.al
        middleware.assay.al
        auth.assay.al
        response.assay.al
        errors.assay.al
```

Every configuration file is generated from the manifest. The test files in `tests/` are scaffolds created by Crucible based on the type signatures it found in `src/`.

:::tip
Run `foundry ignite --dry-run` to preview the full output without writing any files. This is useful for validating a new manifest before committing.
:::

## Step 5: Build and Test

With the workspace forged, you can build and test immediately:

```bash title="Build all packages"
foundry bellows build
```

```bash title="Run all tests"
foundry crucible run
```

Bellows respects the Tongs dependency graph, so `core` builds before `api`. Crucible runs the scaffolded tests — they will pass with placeholder assertions until you fill in the implementation.
