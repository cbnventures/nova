Sync **engines**, **os**, **cpu**, **libc**, **devEngines**, and **packageManager** fields in <Term title="package" to="/docs/quickstart/terminology#nova-concepts">workspace</Term> `package.json` files.

## Summary

The `sync-environment` recipe ensures that runtime environment fields in each <Term title="package" to="/docs/quickstart/terminology#nova-concepts">workspace</Term>'s `package.json` follow consistent conventions. When the `trackNodeLtsVersions` setting is enabled, it automatically updates the `engines.node` constraint to match all currently active Node.js <Term title="long term support" to="/docs/quickstart/terminology#fundamentals">LTS</Term> versions.

## Settings

Settings are configured per workspace in `nova.config.json` under the `recipes` key:

| Setting                | Type    | Description                                                       |
|------------------------|---------|-------------------------------------------------------------------|
| `trackNodeLtsVersions` | boolean | Automatically sync `engines.node` to current Node.js LTS versions |

## Options

| Flag                 | Description                                               |
|----------------------|-----------------------------------------------------------|
| `-d, --dry-run`      | Preview changes without modifying files.                  |
| `-r, --replace-file` | Overwrite the original file instead of creating a backup. |
