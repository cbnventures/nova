# Overview

Use **Nova Rules** to catch unsafe patterns and enforce consistent code style. Pick the rules that match your workflow.

## Install

If the package is not in your repo yet, go to **[Setup and Configure](/docs/quickstart/setup#choose-your-path)** and choose **Install into project**.

:::caution Warning
**Nova** must be installed **in your project** before importing rules. This is because Node.js resolves imports from your project's `node_modules` folder.
:::

:::info
If you wish, you can install Nova **globally** (into the system) and **locally** (into the project) at the same time. They will not conflict with each other.
:::

## Rules

All items below are **targeted**, **independent**, and **safe to adopt**. Enable only what you need. Each rule focuses on a single concern.

### ESLint

Custom rules for JavaScript and TypeScript projects.

#### Formatting

| Rule                                                                         | Synopsis                                                                   |
|------------------------------------------------------------------------------|----------------------------------------------------------------------------|
| [No Raw Text in Code](/docs/rules/eslint/formatting/no-raw-text-in-code)     | Ban unwrapped text inside `<code>` elements to prevent MDX parsing issues. |
| [Require Padding Lines](/docs/rules/eslint/formatting/require-padding-lines) | Require blank lines between specific statement patterns for readability.   |

#### Nova

| Rule                                                   | Synopsis                                                                   |
|--------------------------------------------------------|----------------------------------------------------------------------------|
| [No Logger.dev](/docs/rules/eslint/nova/no-logger-dev) | Flag `Logger.dev` calls so dev-only logging gets caught before production. |

#### Patterns

| Rule                                                                                         | Synopsis                                                                      |
|----------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------|
| [No Assign Then Return](/docs/rules/eslint/patterns/no-assign-then-return)                   | Return expressions directly instead of assigning to an intermediate variable. |
| [No Bracket Assignment](/docs/rules/eslint/patterns/no-bracket-assignment)                   | Use `Reflect.set()` instead of bracket notation assignment.                   |
| [No Destructuring](/docs/rules/eslint/patterns/no-destructuring)                             | Ban destructuring in callbacks, loops, functions, declarations, assignments.  |
| [No Implicit Boolean](/docs/rules/eslint/patterns/no-implicit-boolean)                       | Require explicit comparisons instead of truthy/falsy coercion in conditions.  |
| [No Ternary in Template Literal](/docs/rules/eslint/patterns/no-ternary-in-template-literal) | Extract ternary expressions out of template literal interpolation slots.      |
| [Switch Case Blocks](/docs/rules/eslint/patterns/switch-case-blocks)                         | Require block-wrapped switch cases so declarations stay scoped per branch.    |

#### Regex

| Rule                                                                      | Synopsis                                                                |
|---------------------------------------------------------------------------|-------------------------------------------------------------------------|
| [No Regex Literal Flags](/docs/rules/eslint/regex/no-regex-literal-flags) | Ban flags on regex literals so callers add flags at the call site.      |
| [No Regex Literals](/docs/rules/eslint/regex/no-regex-literals)           | Ban inline regex literals so patterns are centralized in a shared file. |

#### TypeScript

| Rule                                                                                     | Synopsis                                                                     |
|------------------------------------------------------------------------------------------|------------------------------------------------------------------------------|
| [No Catch Unknown Annotation](/docs/rules/eslint/typescript/no-catch-unknown-annotation) | Remove redundant `: unknown` annotations from catch clause variables.        |
| [No Inline Type Annotation](/docs/rules/eslint/typescript/no-inline-type-annotation)     | Require named type aliases instead of inline type annotations in code files. |
| [No Shared Type Import](/docs/rules/eslint/typescript/no-shared-type-import)             | Prevent code files from importing shared type files directly.                |

> Using Nova presets? See **[Presets](/docs/presets/)** for composable ESLint and TSConfig layers.
