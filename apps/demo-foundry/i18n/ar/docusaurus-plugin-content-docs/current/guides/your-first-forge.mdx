# أول طَرق لك

يأخذك هذا الدليل في رحلة لإنشاء مساحة عمل من الصفر باستخدام Foundry. ستكتب بيان `.grain`، وتُشغِّل أول طَرق لك، وتفحص الخرج.

## الخطوة 1: إنشاء دليل المشروع

ابدأ بدليل فارغ:

```bash
mkdir my-project && cd my-project
```

## الخطوة 2: كتابة البيان

أنشئ ملف `project.grain` في الجذر. هذا هو الملف الوحيد الذي يحتاجه Foundry.

```text title="project.grain"
# A minimal workspace with two packages.
workspace "my-project" {
  lang    = "alloy"
  target  = "arcline"
  warden  = ["strict", "conventions"]
  // highlight-start
  crucible = auto
  // highlight-end

  packages {
    core { type = "library" }
    api  { type = "service", depends = ["core"] }
  }
}
```

السطر 6 هو التوجيه الأساسي. ضبط `crucible = auto` يُخبر Foundry بتوليد هياكل اختبار من تواقيع الأنواع تلقائيًا. يمكنك أيضًا ضبطه إلى `manual` إن فضّلت كتابة الاختبارات بنفسك.

## الخطوة 3: تشغيل الطَرق

نفِّذ أمر `foundry ignite`:

```bash title="Forge the workspace"
foundry ignite
```

يقرأ Foundry بيانك، ويحلّ رسم التبعيات، ويُولِّد قطع الإعداد، ويُهيِّكل كل حزمة.


## الخطوة 4: فحص الخرج

بعد الطَرق، يبدو دليلك هكذا:

```text title="Generated workspace structure"
my-project/
  project.grain
  packages/
    core/
      blueprint.grain
      .wardenrc.grain
      alloy.config.grain
      src/
      tests/
        format.assay.al
        validate.assay.al
        parse.assay.al
        transform.assay.al
    api/
      blueprint.grain
      .wardenrc.grain
      alloy.config.grain
      spoke.config.grain
      src/
      tests/
        routes.assay.al
        handlers.assay.al
        middleware.assay.al
        auth.assay.al
        response.assay.al
        errors.assay.al
```

كل ملف إعداد مولَّد من البيان. ملفات الاختبار في `tests/` هي هياكل أنشأها Crucible استنادًا إلى تواقيع الأنواع التي وجدها في `src/`.

:::tip
شغِّل `foundry ignite --dry-run` لمعاينة الخرج الكامل دون كتابة أي ملفات. هذا مفيد للتحقق من بيان جديد قبل الإيداع.
:::

## الخطوة 5: البناء والاختبار

مع مساحة العمل مطروقة، يمكنك البناء والاختبار فورًا:

```bash title="Build all packages"
foundry bellows build
```

```bash title="Run all tests"
foundry crucible run
```

يحترم Bellows رسم تبعيات Tongs، لذا تُبنى `core` قبل `api`. يُشغِّل Crucible الاختبارات المُهيكَلة — ستنجح بتأكيدات مؤقّتة حتى تملأ التنفيذ.
