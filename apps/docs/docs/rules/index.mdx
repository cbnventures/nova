# Overview

Use **Nova Rules** to catch unsafe patterns and enforce consistent code style. Pick the rules that match your workflow.

## Install

If the package is not in your repo yet, go to **[Setup and Configure](/docs/quickstart/setup#choose-your-path)** and choose **Install into project**.

:::warning
**Nova** must be installed **in your project** before importing rules. This is because Node.js resolves imports from your project's `node_modules` folder.
:::

:::info
If you wish, you can install Nova **globally** (into the system) and **locally** (into the project) at the same time. They will not conflict with each other.
:::

## Rules

All items below are **targeted**, **independent**, and **safe to adopt**. Enable only what you need.

Each rule focuses on a single concern.

### ESLint

Custom rules for JavaScript and TypeScript projects.

#### Conventions

| Rule                                                                                          | Synopsis                                                                     |
|-----------------------------------------------------------------------------------------------|------------------------------------------------------------------------------|
| [No Default Export Declaration](/docs/rules/eslint/conventions/no-default-export-declaration) | Disallow attaching function or class declarations to `export default`.       |
| [No Implicit Boolean](/docs/rules/eslint/conventions/no-implicit-boolean)                     | Require explicit comparisons instead of truthy/falsy coercion in conditions. |
| [Require Explicit Return](/docs/rules/eslint/conventions/require-explicit-return)             | Require an explicit `return;` at the end of void function bodies.            |
| [Require Hash Private](/docs/rules/eslint/conventions/require-hash-private)                   | Require `#hash` notation for private class members.                          |
| [Require Kebab Case Filename](/docs/rules/eslint/conventions/require-kebab-case-filename)     | Require kebab-case file names for TypeScript files.                          |
| [Require Naming Convention](/docs/rules/eslint/conventions/require-naming-convention)         | Enforce context-aware naming conventions for identifiers.                    |
| [Require Undefined Init](/docs/rules/eslint/conventions/require-undefined-init)               | Require `= undefined` when declaring uninitialized variables.                |
| [Switch Case Blocks](/docs/rules/eslint/conventions/switch-case-blocks)                       | Require block-wrapped switch cases so declarations stay scoped per branch.   |

#### Formatting

| Rule                                                                                                   | Synopsis                                                                                    |
|--------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------|
| [No Complex Arrow Concise](/docs/rules/eslint/formatting/no-complex-arrow-concise)                     | Require block body on complex concise arrows.                                               |
| [No Multiline Strings](/docs/rules/eslint/formatting/no-multiline-strings)                             | Use `[].join()` instead of multiline string content.                                        |
| [No Raw Text in Code](/docs/rules/eslint/formatting/no-raw-text-in-code)                               | Ban unwrapped text inside `<code>` elements to prevent MDX parsing issues.                  |
| [No Ternary in Template Literal](/docs/rules/eslint/formatting/no-ternary-in-template-literal)         | Extract ternary expressions out of template literal interpolation slots.                    |
| [Require Import Order](/docs/rules/eslint/formatting/require-import-order)                             | Require 6-group import ordering with blank line separation.                                 |
| [Require Multiline Condition Groups](/docs/rules/eslint/formatting/require-multiline-condition-groups) | Require expanded formatting for parenthesized groups in mixed-operator logical expressions. |
| [Require Multiline Conditions](/docs/rules/eslint/formatting/require-multiline-conditions)             | Require multiline format when a logical expression exceeds the inline operand limit.        |
| [Require Padding Lines](/docs/rules/eslint/formatting/require-padding-lines)                           | Require blank lines between specific statement patterns for readability.                    |
| [Require Ternary Parens](/docs/rules/eslint/formatting/require-ternary-parens)                         | Require parenthesized condition in ternaries.                                               |

#### JSDoc

| Rule                                                                                    | Synopsis                                                                           |
|-----------------------------------------------------------------------------------------|------------------------------------------------------------------------------------|
| [Require JSDoc Body](/docs/rules/eslint/jsdoc/require-jsdoc-body)                       | Require a body paragraph in JSDoc blocks that explains why the declaration exists. |
| [Require JSDoc Hierarchy](/docs/rules/eslint/jsdoc/require-jsdoc-hierarchy)             | Require summary lines to follow hierarchy chain from file path.                    |
| [Require JSDoc Param Alignment](/docs/rules/eslint/jsdoc/require-jsdoc-param-alignment) | Require vertical alignment of `@param` types, names, and dashes.                   |
| [Require JSDoc Param Name](/docs/rules/eslint/jsdoc/require-jsdoc-param-name)           | Require `@param` descriptions to match param name.                                 |
| [Require JSDoc Private](/docs/rules/eslint/jsdoc/require-jsdoc-private)                 | Require `@private` tag for private class members.                                  |
| [Require JSDoc Since](/docs/rules/eslint/jsdoc/require-jsdoc-since)                     | Require `@since` tag in every JSDoc block.                                         |

#### Nova

| Rule                                                   | Synopsis                                                                   |
|--------------------------------------------------------|----------------------------------------------------------------------------|
| [No Logger.dev](/docs/rules/eslint/nova/no-logger-dev) | Flag `Logger.dev` calls so dev-only logging gets caught before production. |

#### Patterns

| Rule                                                                                   | Synopsis                                                                                |
|----------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------|
| [No Assign Then Return](/docs/rules/eslint/patterns/no-assign-then-return)             | Return expressions directly instead of assigning to an intermediate variable.           |
| [No Await In Loop](/docs/rules/eslint/patterns/no-await-in-loop)                       | Disallow await inside loops so sequential calls are parallelized.                       |
| [No Boolean Var For If](/docs/rules/eslint/patterns/no-boolean-var-for-if)             | Disallow extracting multi-condition boolean into a variable used once in the next `if`. |
| [No Bracket Assignment](/docs/rules/eslint/patterns/no-bracket-assignment)             | Use `Reflect.set()` instead of bracket notation assignment.                             |
| [No Bracket Method Call](/docs/rules/eslint/patterns/no-bracket-method-call)           | Use dot notation to call methods instead of bracket notation with static keys.          |
| [No Template Curly in String](/docs/rules/eslint/patterns/no-template-curly-in-string) | Catch `${...}` in regular strings so missing backticks are found early.                 |
| [No Use Before Define](/docs/rules/eslint/patterns/no-use-before-define)               | Catch identifiers used before declaration so read order matches definition order.       |

#### Regex

| Rule                                                                      | Synopsis                                                                |
|---------------------------------------------------------------------------|-------------------------------------------------------------------------|
| [No Regex Literal Flags](/docs/rules/eslint/regex/no-regex-literal-flags) | Ban flags on regex literals so callers add flags at the call site.      |
| [No Regex Literals](/docs/rules/eslint/regex/no-regex-literals)           | Ban inline regex literals so patterns are centralized in a shared file. |

#### Safety

| Rule                                                     | Synopsis                                     |
|----------------------------------------------------------|----------------------------------------------|
| [No Script URL](/docs/rules/eslint/safety/no-script-url) | Block `javascript:` URLs in string literals. |

#### Syntax

| Rule                                                                   | Synopsis                                                                          |
|------------------------------------------------------------------------|-----------------------------------------------------------------------------------|
| [No Destructuring](/docs/rules/eslint/syntax/no-destructuring)         | Ban destructuring in callbacks, loops, functions, declarations, assignments.      |
| [No Numeric Literals](/docs/rules/eslint/syntax/no-numeric-literals)   | Use `parseInt()` with an explicit base instead of binary, octal, or hex literals. |
| [No Optional Chaining](/docs/rules/eslint/syntax/no-optional-chaining) | Ban the `?.` operator so null/undefined checks are explicit.                      |
| [No Rest Params](/docs/rules/eslint/syntax/no-rest-params)             | Ban rest parameters so function signatures list parameters explicitly.            |

#### TypeScript

| Rule                                                                                             | Synopsis                                                                     |
|--------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------|
| [No Catch Unknown Annotation](/docs/rules/eslint/typescript/no-catch-unknown-annotation)         | Remove redundant `: unknown` annotations from catch clause variables.        |
| [No Explicit Any](/docs/rules/eslint/typescript/no-explicit-any)                                 | Disallow the `any` type in favor of `unknown` or specific types.             |
| [No Inline Type Annotation](/docs/rules/eslint/typescript/no-inline-type-annotation)             | Require named type aliases instead of inline type annotations in code files. |
| [No Shared Type Import](/docs/rules/eslint/typescript/no-shared-type-import)                     | Prevent code files from importing shared type files directly.                |
| [Require Bracket Property Access](/docs/rules/eslint/typescript/require-bracket-property-access) | Require bracket notation instead of dot notation on project-defined objects. |
| [Require Type Naming](/docs/rules/eslint/typescript/require-type-naming)                         | Require type alias names to start with the class name prefix from file path. |
