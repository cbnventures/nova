# الإعداد

يُعدّ Envoy عبر بيان ترحيل بصيغة `.grain`. يُعلن البيان كلّ مُرحِّل — ما الذي يستمع إليه، وكيف يحوّل الحمولة، وأين يُسلّم، وماذا يحدث حين يفشل التسليم.

## بنية بيان الترحيل

يحتوي بيان الترحيل على كتلة `relay` أو أكثر. كلّ كتلة تُعرّف خطّ معالجة كاملًا من المصدر إلى الوجهة.

```text title="relay.grain"
relay "glassboard-to-canary" {
  source      = "glassboard"
  cipher      = "hmac-sha256"
  destination = "canary://infra-alerts"

  transform {
    title    = "[{{ severity }}] {{ alertname }}"
    body     = "{{ instance }} — {{ message }}"
    priority = severity_to_priority(severity)
    tags     = ["{{ severity_emoji }}"]
  }

  retry {
    strategy = "exponential"
    max      = 5
    backoff  = "1s, 5s, 30s, 2m, 10m"
  }
}

relay "threadbare-to-canary" {
  source      = "threadbare"
  cipher      = "hmac-sha256"
  destination = "canary://ci-builds"

  transform {
    title = "[{{ repo }}] {{ count }} commit(s) to {{ branch }}"
    body  = "{{ author }}: {{ commit_summary }}"
  }
}
```

## توجيهات الإعداد

| التوجيه       | النوع | مطلوب | الوصف                                                                  |
|---------------|-------|-------|------------------------------------------------------------------------|
| `source`      | نصّ   | نعم   | مُعرِّف مصدر التكامل (مثل `threadbare` أو `glassboard`).               |
| `cipher`      | نصّ   | لا    | وضع المصادقة: `hmac-sha256` أو `bearer` أو `ip-allowlist` أو `none`.   |
| `destination` | نصّ   | نعم   | هدف التسليم بصيغة `protocol://topic`.                                  |
| `transform`   | كتلة  | لا    | قواعد تحويل Parcel للحمولة.                                            |
| `retry`       | كتلة  | لا    | سياسة إعادة المحاولات في Courier. الافتراضي تراجع أُسّي بثلاث محاولات. |
| `filter`      | كتلة  | لا    | مُرحِّل شرطي — يعالج فقط الرسائل المطابِقة لتعبير الفلتر.              |
| `fanout`      | قائمة | لا    | التسليم إلى وجهات متعدّدة من مصدر واحد.                                |

## كتلة التحويل

كتلة `transform` تُعرّف كيف يُعيد Parcel كتابة الحمولة. تستخدم الحقول صياغة قوالب على نمط Mustache مع وصول إلى حمولة المصدر.

```text title="Transform directives"
transform {
  title    = "[{{ severity }}] {{ alertname }}"
  body     = "{{ instance }} — {{ message }}"
  priority = severity_to_priority(severity)
  tags     = ["{{ severity_emoji }}"]
  actions  = [
    { label = "View Dashboard", url = "{{ dashboard_url }}" }
  ]
}
```

| الحقل      | النوع          | الوصف                                           |
|------------|----------------|-------------------------------------------------|
| `title`    | قالب           | عنوان الإشعار. يدعم متغيّرات القوالب.           |
| `body`     | قالب           | متن الإشعار. يدعم متغيّرات القوالب وأسطر جديدة. |
| `priority` | رقم/دالّة      | مستوى الأولوية (1-5) أو دالّة تعيين.            |
| `tags`     | قائمة          | مُعرِّفات الوسوم المرتبطة بالإشعار.             |
| `actions`  | قائمة من خرائط | إجراءات تفاعلية (روابط عرض، أزرار رفض).         |

## كتلة إعادة المحاولة

تُعدّ كتلة `retry` سلوك ضمان التسليم في Courier.

```text title="Retry directives"
retry {
  strategy   = "exponential"
  max        = 5
  backoff    = "1s, 5s, 30s, 2m, 10m"
  jitter     = 0.25
  dead_letter {
    enabled    = true
    retention  = "30d"
    escalation = "dispatch://fallback-channel"
  }
}
```

| الحقل         | النوع | الافتراضي     | الوصف                                                               |
|---------------|-------|---------------|---------------------------------------------------------------------|
| `strategy`    | نصّ   | `exponential` | استراتيجية إعادة المحاولة: `exponential` أو `fixed` أو `immediate`. |
| `max`         | رقم   | `3`           | الحدّ الأقصى للمحاولات قبل الرسائل الميّتة.                         |
| `backoff`     | نصّ   | `1s, 4s, 16s` | جدول تأخير مفصول بفواصل لكلّ محاولة.                                |
| `jitter`      | عشري  | `0.25`        | نطاق العشوائية كنسبة من التأخير الحالي.                             |
| `dead_letter` | كتلة  | مُفعَّل       | إعداد طابور الرسائل الميّتة للرسائل غير القابلة للتسليم.            |

## كتلة الفلتر

تُضيف كتلة `filter` منطق ترحيل شرطي. تُعالَج فقط الرسائل المطابِقة لتعبير الفلتر. وتُسقَط البقيّة بصمت.

```text title="Filter example"
relay "critical-alerts-only" {
  source = "glassboard"
  // highlight-start
  filter {
    severity = "critical"
  }
  // highlight-end
  destination = "canary://oncall-urgent"
}
```

## مُرحِّلات متعدّدة

يستطيع بيان واحد إعلان أيّ عدد من المُرحِّلات. يُحمِّلها Envoy جميعها عند البدء ويوجّه الرسائل الواردة إلى أوّل مُرحِّل مطابِق.

```text title="Multi-relay manifest"
relay "critical-glassboard"  { source = "glassboard", filter { severity = "critical" }, destination = "canary://oncall-urgent" }
relay "standard-glassboard"  { source = "glassboard", destination = "canary://infra-alerts" }
relay "threadbare-pushes"    { source = "threadbare", destination = "canary://ci-builds" }
relay "canary-heartbeats"    { source = "canary", destination = "canary://monitoring" }
```

:::warning ترتيب المُرحِّلات
تُقيَّم المُرحِّلات بترتيب الإعلان. إن طابق مُرحِّلان المصدر نفسه، يفوز الأوّل. ضع المُرحِّلات الأكثر تخصيصًا (التي تحوي فلاتر) قبل المُرحِّلات العامّة الجامعة.
:::

## التحقّق

تحقّق من البيان قبل بدء المُرحِّل:

```bash title="Validate configuration"
envoy validate --config relay.grain
```

```text title="Validation output"
[validate] Loading relay.grain
[validate] Relay "glassboard-to-canary" ... OK
[validate] Relay "threadbare-to-canary" ... OK
[validate] 2 relays configured, 0 errors
[validate] Configuration is valid.
```
