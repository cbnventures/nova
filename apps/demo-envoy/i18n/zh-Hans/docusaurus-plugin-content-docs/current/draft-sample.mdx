# 草稿样例

下一轮路由回退迭代的工作笔记。当投递目标连续拒绝三次时，我们目前会把消息烧入死信环，并触发值班呼叫。

我们在考虑是否存在一条更柔和的中间路径——一个熔断器，它会先把通道切换到"保留并重放"状态九十秒，再决定是否呼叫值班；保留期间本地缓冲，发送方完全感受不到这道缝隙。

在它离开笔记本之前，需要先与当前的 SLO 预算（p95 八分钟）做一次对比。此外还有一个待解决的问题：在乱序投递通道上，这种保留是否会引入顺序风险。目前都还没有定论。
