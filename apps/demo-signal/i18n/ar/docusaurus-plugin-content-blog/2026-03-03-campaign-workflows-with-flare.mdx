اعتاد إطلاق حملة متعددة القنوات أن يشمل أربع عشرة خطوة يدوية عبر خمس أدوات. إنشاء روابط التتبع في أداة. بناء جدول UTM في أخرى. إعداد صيغ البريد. ضبط منشورات التواصل. توصيل لوحات التحليلات. نسخ ولصق معرّفات الروابط بين الأنظمة. الدعاء بألا يُساء كتابة شيء.

رسمنا العملية من البداية إلى النهاية ووجدنا أن إطلاق حملة وسيطة يستغرق 4.5 ساعة من عمل التنسيق، لا عمل إبداعي ولا استراتيجية، مجرّد توصيل. خفض Flare ذلك إلى 22 دقيقة.

## العملية القديمة

إليك كيف بدا إطلاق حملة بخمس قنوات قبل Flare:

1. صياغة موجز الحملة في مستند.
2. إنشاء معاملات UTM في جدول بيانات.
3. توليد روابط Beacon لكل تركيبة قناة-صيغة.
4. نسخ معرّفات روابط Beacon إلى جدول UTM.
5. إنشاء قوالب بريد بروابط Beacon مضمَّنة.
6. إنشاء مسودات منشورات تواصل بروابط Beacon مضمَّنة.
7. ضبط عناوين URL للإعلانات المدفوعة بروابط Beacon الصحيحة.
8. إعداد رموز QR لمواضع الطباعة.
9. بناء لوحة Pulse للحملة.
10. ضبط تنبيهات Pulse.
11. إنشاء شريحة Lens للجمهور المستهدف.
12. تقديم الروابط لمراجعة الامتثال.
13. جدولة الحملة عبر كل قناة.
14. التحقق من أن جميع الروابط تُحلَّ صحيحاً.

الخطوات من 2 إلى 11 كانت تنسيقاً صرفاً. كل واحدة تتضمن تبديل أدوات ونسخ ولصق معرّفات وضمان اتساق يدوي عبر الأنظمة. خطأ إملائي واحد في الخطوة 4 كان قادراً على إرسال نقرات قناة كاملة إلى دلو إسناد خاطئ.

## ماذا يفعل Flare

Flare طبقة تنسيق حملات تجلس فوق Beacon وPrism وPulse وLens وRelay. تُعرّف الحملة مرة، ويتولى Flare توليد الروابط ووسم UTM وإعداد اللوحة وضبط التنبيهات واستهداف الجمهور:

```json title="تعريف حملة Flare"
{
  "campaign": "core-activation-trial-004",
  "channels": ["email", "social", "paid", "qr-print"],
  "variants": {
    "email": ["hero-cta", "sidebar-link", "footer-banner"],
    "social": ["og-card", "thread-link"],
    "paid": ["search-brand", "display-retarget"],
    "qr-print": ["conference-badge", "booth-banner"]
  },
  "audience": {
    "segment": "high-intent-evaluators",
    "source": "lens"
  },
  "monitoring": {
    "dashboard": true,
    "alerts": [
      { "metric": "ctr", "below": 0.015, "window": "1h" },
      { "metric": "conversionRate", "below": 0.01, "window": "4h" }
    ]
  }
}
```

من هذا التعريف، يولّد Flare:

- **9 روابط Beacon** (واحد لكل تركيبة قناة-صيغة) بمعاملات UTM متّسقة.
- **لوحة Pulse واحدة** مضبوطة مسبقاً بالروابط التسعة، مجمَّعة بالقناة.
- **تنبيهان Pulse** بالعتبات المُعرَّفة في مواصفة الحملة.
- **استعلام شريحة Lens واحدة** مرتبط بالحملة لتحليل الجمهور.
- **مهمة Relay واحدة** جاهزة للتنفيذ إن احتاجت تعديلات روابط بالجملة في منتصف الحملة.

## مقارنة الوقت

| الخطوة            | يدوي       | Flare        |
|-------------------|------------|--------------|
| توليد الروابط     | 35 دقيقة   | 0 دقيقة      |
| ضبط UTM           | 20 دقيقة   | 0 دقيقة      |
| إعداد اللوحة      | 25 دقيقة   | 0 دقيقة      |
| ضبط التنبيهات     | 15 دقيقة   | 0 دقيقة      |
| استهداف الجمهور   | 20 دقيقة   | 0 دقيقة      |
| تعريف الحملة      | 0 دقيقة    | 12 دقيقة     |
| المراجعة والإطلاق | 45 دقيقة   | 10 دقائق     |
| التحقق من الروابط | 30 دقيقة   | 0 دقيقة      |
| **الإجمالي**      | **4س 30د** | **22 دقيقة** |

لا يقصي Flare العمل الإبداعي. يقصي التوصيل. تُقضى الـ 22 دقيقة المتبقية على تعريف الحملة (وهو الاستراتيجية) والمراجعة النهائية (وهي فحص الجودة).

## التعديلات في منتصف الحملة

كانت العملية القديمة أسوأ للتغييرات في منتصف الحملة. تحديث عنوان URL صفحة هبوط عبر جميع الروابط التسعة كان يعني تحرير كل واحد فردياً، والتحقق من كل إعادة توجيه، وتحديث جدول التتبع. يتعامل Flare مع الحملة ككائن واحد، تغيير وجهة يتدفق إلى كل رابط في الحملة عبر Relay.

الحملة ليست مجموعة روابط. إنها نظام منسَّق. يتعامل معها Flare كذلك.
